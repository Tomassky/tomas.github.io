Kali Linux笔记(七十六)：封装TCP流量隧道--stunnel4

发表于 2020-02-03 | 分类于 Kali
字数统计: 569 字 | 阅读时长 ≈ 2 分钟

1. stunnel4

  • 无需修改原代码的情况下将TCP流量封装于SSL通道内
  • 适用于本身不支持加密传输的应用
  • 支持openssl安全特性
  • 跨平台
  • 性能优

2. 实验

  • Mono防火墙
    • 第一个网卡为Hosy-only方式，第二个网卡为Bridge桥接方式，桥接到无线网络，检查
  • bodhi（stunnel4 Server）
    • 网卡方式为NAT方式，检查网络
  • metasploit（内网的应用服务器，mysql）
    • 网卡方式为Host-only方式，检查网络
  • bodhi（stunnel4 Server）
    • 🔺命令：apt-get install stunnel4
    • 网卡方式修改为Host-only方式，检查网络
    • 浏览器访问mono防火墙
    • 编辑上一个小节的NAT规则
      • Protocol → TCP
      • External port range → from:3306
      • NAT IP → Metasploit的IP地址
      • Local port → 3306
      • 保存更改的应用
    • 编辑上一小节的Rules规则（WAN口）
      • Destination port range → from:other 3306/to:3306
      • 保存更改的应用
  • Kali（stunnel4 Client）
    • 网卡方式为Bridge方式，桥接到无线网络，检查网络
    • 开启Wireshark抓包（ip.address=mono的WAN的IP）
    • 🔺命令：mysql -u root -h mono的WAN的IP
    • 🔺命令：show databases;
    • ▲ Wireshark的TCP Stream可以查看到为明文信息
  • bodhi（stunnel4 Server）
    • 生成证书：🔺命令：openssl req -new –days 365 -nodes -x509 -out /etc/stunnel/ stunnel.pem -keyout /etc/stunnel/stunnel.pem
    • 创建配置文件：/etc/stunnel/stunnel.conf
      • cert = /etc/stunnel/stunnel.pem
      • setuid = stunnel4
      • setgid = stunnel4
      • pid = /var/run/stunnel4/stunnel4.pid
      • [mysqls]
      • accept = 0.0.0.0:443
      • connect = Metasploit的IP:3306
    • stunnel4自动启动
      • /etc/default/stunnel4 → ENABLED=1
    • 启动stunnel4服务端
      • 🔺命令：service stunnel4 start（netstat -pantu进行查看是否开启）
    • 浏览器登陆mono修改WAN的NAT
      • External port range → HTTPS/HTTPS
      • NAT IP → bodhi的IP
      • 保存应用
    • 浏览器登陆mono修改WAN的Rule规则
      • Destination → Address → bodhi的IP
      • Destination port range → HTTPS/HTTPS
      • 保存应用
  • Kali（stunnel4 Client）
    • 创建配置文件：/etc/stunnel/stunnel.conf
      • client = yes
      • [mysqls]
      • accept = 3306
      • connect = mono的IP:443
    • 客户端自动启动
      • /etc/default/stunnel4 → ENABLED=1
    • 启动stunnel4客户端
      • 🔺命令：service stunnel4 stop / start（netstat -pantu进行查看是否开启）
    • 开启Wireshark抓包（ip.address=mono的IP）
    • 🔺命令：mysql -u root -h 127.0.0.1
    • 🔺命令：show databases;
    • ▲ Wireshark的TCP Stream可以查看到为加密信息
  • ▲ 2020.2.3外网WAN访问防火墙WAN的IP，无法进行访问，进行ping防火墙WAN的IP也无法成功，猜测与防火墙规则有关，后面有关stunnel的实验都未作，猜测VMware自带网卡的问题

Kali Linux笔记(七十五)：一端口多服务隧道--sslh

发表于 2020-02-03 | 分类于 Kali
字数统计: 906 字 | 阅读时长 ≈ 3 分钟

1. sslh（ssl+ssh）

  • 端口分配器
    • 根据客户端第一个包检测协议类型
    • 根据协议检测结果将流量转发给不同目标
    • 支持HTTP、HTTPS、SSH、OpenVPN、tinc、XMAPP和可基于正则表达式判断的人和协议类型
    • 适用于防火墙允许443端口入站访问流量的环境

2. 实验

  • Mono防火墙
    • 第一个网卡为Hosy-only方式，第二个网卡为Bridge桥接方式，桥接到无线网络，检查
  • Kali（SSLH）
    • 网卡方式为Host-only方式，检查网络
  • Window Server 2003（内网的CA服务器）
    • 网卡方式为Host-only方式，检查网络
    • 安装web server
      • Win+R → appwiz.cpl → 更改或添加程序 → 应用程序服务器 → IIS → 万维网服务/Active Server Pages，可能需要光盘）
    • 安装CA服务器
      • Win+R → appwiz.cpl → 更改或添加程序 → 证书服务
      • 独立根CA类型、证书名称：LAB CA、有效期限
    • 开始 → Internet信息服务（IIS）管理器
      • w2k3（本地计算机）→ 网站 → 默认网站 → 右键 → 属性
      • 目录安全性 → 服务器证书 → 名称：www.lab.com、2048位长、单位：IT、部门：IT、公用名称：www.lab.com、国家/省市 → 修改申请文件放置位置
      • 浏览器范围访问http://127.0.0.1/certsrv/ → 申请证书 → 高级证书 → BASE64编码 → 粘贴证书服务器代码 → 提交
    • 开始 → 证书服务器管理
      • LAB CA → 挂起的申请 → 选中申请的证书右键 → 所有任务 → 颁发
      • 证书可以在颁发的证书中查看详细信息
    • 浏览器范围访问http://127.0.0.1/certsrv/ → 查看挂起的证书申请状态 → 下载证书 → 保存在桌面
    • 开始 → Internet信息服务（IIS）管理器
      • w2k3（本地计算机）→ 网站 → 默认网站 → 右键 → 属性
      • 目录安全性 → 服务器证书 → 处理挂起的请求并安装证书
      • w2k3（本地计算机）→ 网站 → 默认网站 → 右键 → 属性
      • 目录安全性 → 查看证书
      • w2k3（本地计算机）→ 网站 → 默认网站 → 右键 → 属性
      • 目录安全性 → 编辑 → 要求安全通道（SSL）
      • 停止网站 → 重新启动
  • Kali（SSLH）
    • 浏览器访问https://Window Server 2003的IP，看是否能通过SSL通道进行通信
    • 配置文件：🔺命令：vi /etc/default/sslh
      • RUN=yes
      • DAEMON=/usr/sbin/sslh
      • DAEMON_OPTS=”–user sslh –listen 本机的IP:443 –ssh 127.0.0.1:22 –http 127.0.0.1:80 –ssl Window Server 2003的IP:443 –pidfile /var/run/sslh/sslh.pid”
    • 启动apache服务，修改侦听端口🔺命令：vi /etc/apache2/ports.conf → 127.0.0.1:80，重启apache服务，🔺命令：netstat -pantu查看是否更改
    • 开启SSH服务（修改SSH配置文件）
  • Mono防火墙
    • 浏览器访问mono防火墙
    • 新建NAT规则
      • Interface → WAN
      • External address → Interface address（若为网段其他的IP映射，需设置为WAN address → Server NAT → 加入整个网段）
      • Protocol → TCP
      • External port range → HTTPS/HTTPS
      • NAT IP → KALI的IP
      • Auto-add a firewall rule to permit traffic through this NAT rule
    • 保存应用，Rules的WAN口多出一条rule
  • Kali（SSLH）
    • 🔺命令：service sslh start（netstat -pantu进行查看）
    • 浏览器验证本机http是否正常进行
  • 物理机
    • 浏览器访问http://防火墙WAN的IP:443 → Kali的Web资源
    • 浏览器访问https://防火墙WAN的IP → Window Server 2003的Web资源
    • ssh尝试登陆防火墙WAN的IP:443进行登陆
  • 🔺命令：namp -p443 -sV -A 防火墙WAN的IP
    • 基本可以扫描出所有的服务
  • ▲ 2020.2.3内网sslh代理正常，原因在于访问Kali IP三种方式都能代理，但外网WAN访问防火墙WAN的IP，无法进行访问，进行ping防火墙WAN的IP也无法成功，猜测与防火墙规则有关

Kali Linux笔记(七十四)：HTTP(S)隧道--proxyptunnel

发表于 2020-02-03 | 分类于 Kali
字数统计: 1.2k 字 | 阅读时长 ≈ 4 分钟

1. proxytunnel

  • 通过标准的HTTP/HTTPS代理创建隧道的工具
  • 通过HTTP CONNECT方法封装信息
  • 适用于内网使用代理并且防火墙只允许代理服务器上网的场景
  • 无法创建DNS隧道和ICMP隧道
  • 侦听端口开在本地上，可以指定需要访问的资源，服务器

2. 实验-1–准备

  • 将外网资源映射为内网指定端口
  • Mono防火墙
    • 第一个网卡为Hosy-only方式，第二个网卡为Bridge桥接方式，桥接到无线网络，检查
  • bodhi（代理服务器）
    • 网卡方式为Host-only方式，检查网络
  • PC（bodhi克隆）
    • 网卡方式为Host-only方式，检查网络
  • Kali
    • 网卡方式为Bridge方式，桥接到无线网络，检查网络

3. 实验-1

  • bodhi（代理服务器）
    • 浏览器登陆mono防火墙，增加规则只允许HTTP80访问，LAN -> WAN HTTP
    • 暂时开启mono防火墙默认规则，方便下squid3和proxytunnel
    • 🔺命令：sudo apt-get install squid3（默认所有IP监听3128端口）
    • 🔺命令：vi /etc/squid3/squid.conf ▲ 修改侦听IP，防止公网连接
      • /^http_port → 内网IP:3128
      • /^http_access → 修改deny为allow（ports/ssl/all等）
      • /^acl → 查看安全策略
    • 🔺命令：sudo squid3 -k kill（会自动重启）
    • 🔺命令：netstat -pantu | grep 3128（查看是否限制为监听内网的3128端口）
  • PC（bodhi克隆）
    • 🔺命令：sudo apt-get install proxytunnel
  • bodhi（代理服务器）
    • 关闭mono防火墙默认规则
  • PC（bodhi克隆）
    • 在无代理的情况下访问外网资源尝试
    • 设置HTTP proxy → 代理服务器IP:3128，进行访问外网资源尝试
  • Kali
    • 开启Web服务
    • 启动Wireshark抓包
  • PC（bodhi克隆）
    • 访问Kali的Web服务进行尝试
  • Kali
    • 查看抓包的数据分析（TCP Stream → Via/X-Fowarded-For会泄露真实IP以及代理IP）
  • PC（bodhi克隆）
    • 关闭代理端口
    • 🔺命令：proxytunnle -a（监听端口）80 -p 代理服务器IP:3128 -d Kali的IP:80
    • 浏览器访问http:127.0.0.1:80进行尝试

4. 实验-2–准备

  • 外网资源非防火墙允许端口
  • 修改目标资源侦听端口可能无法躲避防火墙深层检测（协议检测可发现非80HTTP流量）
  • 外网安装squid3代理服务器并侦听80端口
  • Mono防火墙
    • 第一个网卡为Hosy-only方式，第二个网卡为Bridge桥接方式，桥接到无线网络，检查
  • bodhi（代理服务器）
    • 网卡方式为Host-only方式，检查网络
  • PC（bodhi克隆）
    • 网卡方式为Host-only方式，检查网络
  • Kali（代理服务器+目标资源）
    • 网卡方式为Bridge方式，桥接到无线网络，检查网络

5. 实验-2

  • bodhi（代理服务器）
    • 浏览器登陆mono防火墙，增加规则只允许HTTP80访问，LAN -> WAN HTTP
    • 暂时开启mono防火墙默认规则，方便下squid3和proxytunnel
    • 🔺命令：sudo apt-get install squid3（默认所有IP监听3128端口）
    • 🔺命令：vi /etc/squid3/squid.conf ▲ 修改侦听IP，防止公网连接
      • /^http_port → 内网IP:3128
      • /^http_access → 修改deny为allow（ports/ssl/all等）
      • /^acl → 查看安全策略
    • 🔺命令：sudo squid3 -k kill（会自动重启）
    • 🔺命令：netstat -pantu | grep 3128（查看是否限制为监听内网的3128端口）
  • PC（bodhi克隆）
    • 🔺命令：sudo apt-get install proxytunnel
  • bodhi（代理服务器）
    • 关闭mono防火墙默认规则
  • PC（bodhi克隆）
    • 在无代理的情况下访问外网资源尝试
    • 设置HTTP proxy → 代理服务器IP:3128，进行访问外网资源尝试
  • Kali（代理服务器+目标资源）
    • 关闭apache服务
    • 🔺命令：apt-get install squid3
    • 🔺命令：vi /etc/squid3/squid.conf
      • /^http_port → 本机IP:80
      • /^http_access → 修改deny为allow（ports/ssl/all等）
      • /^acl → 查看安全策略
    • 🔺命令：service squid3 start（netstat -pantu查看）
    • 开启SSH服务（修改SSH配置文件，netstat -pantu查看）
    • 开启Wireshark抓包（主要抓取888）
  • PC（bodhi克隆）
    • 🔺命令：proxytunnle -a 2222 -p 代理服务器IP:3128 -r Kali的IP:80 -d Kali的IP:22
    • 🔺命令：ssh root@127.0.0.1 -p 2222进行连接尝试（密钥交换、输入密码 → ifconfig）
    • 🔺命令：proxytunnle -a 888 -p 代理服务器IP:3128 -r Kali的IP:80 -d 192.168.0.1:80
    • 浏览器访问127.0.0.1:888
  • Kali
    • ▲ Wireshark抓包可看出，TCP Stream（tcp.stream eq 0）→ 方法为CONNECT
  • ▲ 2020.2.3数据包能抓到CONNECT的流量，但建立的SSH和HTTP都无法访问，数据包显示为Method Not Allowed方法不允许

6. 实验-3

  • ssh客户端配置自动创建代理链隧道
  • PC（bodhi克隆）
    • 🔺命令：sudo vi ~/.ssh/config
      • Host Kali的IP
      • Hostname Kali的IP
      • ProtocolKeepAlives 30
      • ProxyCommand /usr/bin/proxytunnel \
      • -p 代理服务器IP:3128 \
      • -r Kali的IP:80 -d %h:%p
    • 🔺命令：ssh root@Kali的IP -p 22
  • ▲ 2020.2.3数据包能抓到CONNECT的流量，但建立的SSH无法访问，数据包显示为Method Not Allowed方法不允许

Kali Linux笔记(七十三)：ICMP隧道--ptunnel

发表于 2020-02-02 | 分类于 Kali
字数统计: 427 字 | 阅读时长 ≈ 1 分钟

1. ptunnle

  • Ping tunnel ICMP隧道工具
    • 通过ICMP echo（ping requests，8）和reply（ping reply，0）实现隧道
    • 适用于防火墙只允许ping出站流量的环境
    • 支持多并发连接、性能优
    • 支持身份验证
    • 需要root权限
    • 支持抓包
      • Win：winpcap
      • Linux：libpcap
    • 工作过程（指定Proxy的时候，需要指定目的地，客户端指定）
      • Proxy、Client、Destination
  • 服务器
    • 🔺命令：ptunnel -x 1234
  • 客户端
    • 🔺命令：sudo ptunnel -p proxy -lp 2222 -da destination -dp 22 -x 1234
  • 嵌套SSH隧道
    • 🔺命令：ssh -CNfg -D 7000 root@127.0.0.1 -p 2222
  • ▲ ptunnel直到目前的最新版仍存在拒绝服务漏洞
    • 0.72

2. 实验环境准备

  • Kali（Proxy&Destinatiom）
    • 网卡方式为Bridge方式，桥接到无线网络，检查网络
  • Mono防火墙
    • 第一个网卡为Hosy-only方式，第二个网卡为Bridge桥接方式，桥接到无线网络，检查
  • Kali（内网）
    • 网卡方式为Host-only方式，检查网络
    • 浏览器访问mono，检查防火墙规则
      • Protocol → ICMP
      • Sourece → LAN subnet
      • LAN DNS -> WAM PING
    • ping/上网尝试
  • Window XP SP3
    • 网卡方式为Host-only方式，检查网络

3. 实验

  • Kali（Proxy&Destinatiom）
    • 服务器：🔺命令：ptunnel -x 1234
    • 启动Wireshark抓包，查看发送的数据包内容（ICMP数据包）
  • Kali（内网）
    • 🔺命令：sudo ptunnel -h
    • 客户端：🔺命令：sudo ptunnel -p Kali的IP -lp 2222 -da 192.168.0.1 -dp 80 -x 1234
    • 浏览器访问127.0.0.1:2222进行尝试
  • Kali（Proxy&Destinatiom）
    • 开启SSH（修改SSH配置文件）
  • Kali（内网）
    • 🔺命令：sudo ptunnel -p Kali的IP -lp 2222 -da Kali的IP -dp 22 -x 1234
    • 🔺命令：ssh -CNfg -D 7000 root@127.0.0.1 -p 2222
  • Window XP SP3
    • 设置浏览器Internet代理，用内网Kali的IP:7000，上网尝试

Kali Linux笔记(七十二)：隧道传输工具--SOCAT

发表于 2020-02-02 | 分类于 Kali
字数统计: 763 字 | 阅读时长 ≈ 3 分钟

1. SOCAT

  • 被称为nc++（增强版的nc）
    • 双向数据流通道工具
  • 连接端口
    • 🔺命令：socat -（标准的输入输出） tcp:1.1.1.1:80（需要连接的目标）
  • 侦听端口
    • 🔺命令：socat - tcp4-listen:22 / socat - tcp-l:33
  • 接受文件
    • 🔺命令：socat tcp4-listen:333 open:2.txt,creat,append
  • 发送文件
    • 🔺命令：cat 1.txt | socat - tcp4:1.1.1.1:333
  • 远程shell–服务器端
    • 🔺命令：socat tcp-l:23 exec:sh,pty,stderr
  • 端口转发
    • 🔺命令：socat tcp4-listen:22,fork tcp4:1.1.1.1:22
  • 远程执行命令
    • 服务器：🔺命令：socat - udp-l:2001
    • 客户端：🔺命令："id" | socat - udp4-datagram:1.1.1.1:2001
  • UDP全端口任意内容发包
    • 🔺命令：for PORT in {1..65535}; do echo “aaaaa" | socat - UDP4-DATAGRAM: 1.1.1.1:$PORT; sleep .1; done
  • 二进制编辑器
    • 🔺命令：echo -e "\0\14\0\0\c" | socat -u - file:/usr/bin/ squid.exe,seek,seek=0x00074420
  • 简单的WEB服务
    • 🔺命令：socat -T 1 -d -d TCP-L:10081,reuseaddr,fork,crlf SYSTEM:"echo -e \"\\ \"HTTP/1.0 200 OK\\\nDocumentType: text/plain\\\n\\\ndate: \$\(date\)\\ \nserver:\$SOCAT_SOCKADDR:\$SOCAT_SOCKPORT\\\nclient: \ $SOCAT_PEERADDR:\$SOCAT_PEERPORT\\\n\\\"\"; cat; echo -e \"\\\"\\\n\ \\"\""

2. 实验环境准备

  • Kali
    • 网卡方式为Bridge方式，桥接到无线网络，检查网络
  • bodhi
    • 网卡方式为Bridge方式，桥接到无线网络，检查网络

3. TCP连接/侦听端口

  • Kali
    • 🔺命令：socat - tcp:192.168.0.1:80
    • 🔺命令：get / http 1.1/
    • 🔺命令：head
    • 侦听端口：🔺命令：socat - tcp4-listen:2222（作为服务器端使用）
  • bodhi
    • 🔺命令：apt-get install socat
    • 🔺命令：socat - tcp:Kali的IP:2222
    • 进行传输数据测试

4. TCP传送/接收文件

  • Kali
    • 🔺命令：socat tcp4-listen:333 open:2.txt,creat,append（接收文件）

2. Dns2tcp

  • 利用合法的DNS服务器实现DNS隧道
  • C/S（dns2tcpc/dns2tcpd）结构

  • 域名记录长度是有限的

Kali Linux笔记(六十七)：SSH隧道--远程端口转发

发表于 2020-02-01 | 分类于 Kali
字数统计: 649 字 | 阅读时长 ≈ 2 分钟

1. SSH远程端口转发

  • 由于ACL等原因，SSH与应用连接建立方向相反
  • 本地端口转发
    • SSH客户端+应用客户端位于防火墙一端
    • SSH服务端+应用服务端位于另一端
  • 远程端口转发
    • SSH客户端、应用客户端位于防火墙两端
    • SSH服务端、应用服务端位于防火墙两端
  • 之所以称为远程，是因为SSH侦听端口开在远程的SSH Server服务上
  • 侦听端口永远开在应用客户端一方
  • ▲ 远程端口转发不支持网管模式

2. 准备

  • 实验机子：Linux主机/防火墙（mono）/Web服务器（KALI/2007），关闭机子的防火墙
