Git笔记-5：Git的回滚撤销

发表于 2020-02-20 | 分类于 Git
字数统计: 324 字 | 阅读时长 ≈ 1 分钟

1. Git的回滚撤销

  • 返回前一次的记录：🔺命令：git reset master^/~5/hash值id
    • ^：每一个^符号标识[前一次]，如^^^表示回退三次
    • ：回退次数加在后面，如5
    • hash值id：回退到id号的版本
    • –mix：把暂存区的文件丢弃，但不会动到工作目录的文件（默认方式）
    • –soft：工作目录跟暂存区的文件都不会被丢弃
    • –hard：不管是工作目录以及暂存区的文件都会丢弃
  • 所有的日志文件：🔺命令：git reflog
  • 撤销某次操作：🔺命令：git revert
    • 此次操作之前和之后的commit和history都会保留，并且把这次撤销作为一次最新的提交
  • ▲ 在公用分支上应用revert，在私用分支上用reset
reset和revert
revert：用一次新的commit来回滚之前的commit
reset：只是将旧的commit指针移动，并无删除旧的commit

1. Git的合并

  • ▲ 可以在Stackoverflow上看git log多样格式
  • ▲ rebase的黄金法则：绝对不要在公共分支使用rebase
  • 查看日志：git log
    • –oneline：一行的提交信息
    • –oneline -3：只取前面3行
  • 查看添加的内容：🔺命令：git show 哈希值
  • 查看提交方式：🔺命令：git merge --help
    • –ff（fast-forward）：默认不产生一次commit
    • –no-ff：产生一个commit
  • 移动分支，并将master分支的提交并入：🔺命令：git rebase
  • 查看版本分支报错情况：🔺命令：git mergetool
    • 通过工具修改后，会生成一个x.txt.orig，保存冲突的现场

merge和rebase
merge：两个分支的修改提交，默认不提交，关注点在合并操作上
rebase：将当前分支做的修改，复制在目标分支的最后一次中，关注点在开发的过程上

Git笔记-1：什么是Git

发表于 2020-02-20 | 分类于 Git
字数统计: 279 字 | 阅读时长 ≈ 1 分钟

1. Git

  • 分布式版本管理系统
    • 最初是为了管理Linux的系统内核
    • 同FTP相同的功能
    • 非线性管文件
    • 速度快
    • 对待数据更像一个快照流
    • 可以在无网络的情况下进行提交到本地服务器，在有网络的情况下提交给远端的服务器

  • 方式：直接记录快照而非方式，每次修改需要一次Commit
  • 保证完整性：计算校验和（哈希算法），Git存取的是哈希值

2. Git中的数据库

  • Git仓库–.git directory：保存项目数据，元数据的地方，git自己维护的文件夹
  • 工作目录–Working Directory：内容自Git仓库，对当前项目某一版本独立提取出来的内容
  • 暂存区–Staging Area：保存下次提交的文件目录，也算是一个索引
  • 三个概念
    • 已提交：Git目录中保存着的特定版本文件
    • 已暂存：做了修改并放在暂存区
    • 已修改：做了修改但还没有放到暂存区
  • 远程仓库–Remote：多人操作的仓库

渗透工具(一)：Dirsearch

发表于 2020-02-18 | 分类于 渗透工具
字数统计: 458 字 | 阅读时长 ≈ 2 分钟

1. 简介

  • 目录爆破工具–dirsearch is a simple command line tool designed to brute force directories and files in websites.
  • 语言：Python3
  • 平台
    • Windows XP/7/8/10
    • GNU/Linux
    • MacOSX
  • 官方声明特性
    • Multithreaded：多线程运行
    • Keep alive connections：长连接
    • Support for multiple extensions (-e|–extensions asp,php)：多文件格式
    • Reporting (plain text, JSON)：报告
    • Heuristically detects invalid web pages：启发式检测无效的web页面
    • Recursive brute forcing：递归式暴力破解
    • HTTP proxy support：http代理支持
    • User agent randomization：随机的user-agnet
    • Batch processing：批量处理
    • Request delaying：请求延时
    • ▲ 可自建字典
  • 下载
    • Github地址：https://github.com/maurosoria/dirsearch
    • git clone https://github.com/maurosoria/dirsearch
    • cd dirsearch/
    • ./dirsearch.py
  • 注意事项
    • 不支持除http代理外的其他代理，如sock5
    • 批量处理可能导致服务器拒绝服务，线程数不超过10

2. 常用参数

  • -h, –help 查看帮助
  • Mandatory
    • -u URL, –url=URL 设置url
    • -L URLLIST, –url-list=URLLIST 设置url列表
    • -e EXTENSIONS, –extensions=EXTENSIONS 网站脚本类型
  • Dictionary Settings
    • -w WORDLIST, –wordlist=WORDLIST 设置字典
    • -l, –lowercase 小写
    • -f, –force-extensions 强制扩展字典里的每个词条
  • General Settings
    • -s DELAY, –delay=DELAY 设置请求之间的延时
    • -r, –recursive Bruteforce recursively 递归扫描
    • -t THREADSCOUNT, –threads=THREADSCOUNT 设置扫描线程
    • -x EXCLUDESTATUSCODES, –exclude-status=EXCLUDESTATUSCODES 排除指定的网站状态码（用逗号隔开）
    • -c COOKIE, –cookie=COOKIE 设置cookie
    • –ua=USERAGENT, –user-agent=USERAGENT 设置用户代理
    • -F, –follow-redirects 跟随地址重定向扫描
    • -H HEADERS, –header=HEADERS 设置请求头（Referer等）
    • –random-agents, –random-user-agents 设置随机代理
  • Connection Settings
    • –timeout=TIMEOUT 设置超时时间
    • –ip=IP 设置域名解析的IP地址
    • –proxy=HTTPPROXY, –http-proxy=HTTPPROXY 设置http代理（例如localhost:8080）
      –max-retries=MAXRETRIES 设置最大的重试次数
  • Report
    • –simple-report=SIMPLEOUTPUTFILE 保存结果，发现的路径
    • –plain-text-report=PLAINTEXTOUTPUTFILE 保存结果，发现的路径和状态码
    • –json-report=JSONOUTPUTFILE 以json格式保存结果

Kali Linux笔记(九十七)：Metasploit Framework

发表于 2020-02-17 | 分类于 Kali
字数统计: 613 字 | 阅读时长 ≈ 2 分钟

1. Metasploit简介

  • 目前最流行、最强大、最具扩展性的渗透测试平台软件
  • 基于Metasploit进行渗透测试和漏洞分析的流程和方法
  • 2003年由HD More发布第一版，2007年用ruby语言重写
    • 框架继承了渗透测试标准（PETS）思想
    • 一定程度上统一了渗透测试和漏洞研究的工作环境
    • 新的攻击代码可以比较容易的加入框架
  • 开发活跃版本更新频繁（每周）
    • 早期版本基于社区力量维护，被Rapid 7收购后打造出其商业版产品
    • 目前分化为四个版本，社区版本依然十分活跃
    • HD More说：为Metasploit写书是种自虐！
    • 2014年之后市场上没有再出现新的Metasploit教材
  • MSF默认集成与Kali Linux之中
  • 使用postgresql数据库存储数据
    • 早期版本需要先启动数据库再启动msf

2. Offensive security出版的Metasploit教材

  • 被HD More称之为当时最好的Metasploit教材（2011/2012）

3. 版本对比

  • Pro版是企业级全功能的高级渗透测试平台

4. MSF架构

  • Rex
    • 基本功能库，用于完成日常基本任务，无需人工手动编码实现
    • 处理socket连接访问、协议应答（http/SSL/SMB等）
    • 编码转换（XOR、Base64、Unicode）
  • Msf::Core
    • 提供Msf的核心基本API，是框架的核心能力实现库
  • Msf::Base
    • 提供友好的API接口，便于模块调用的库
  • Plugin插件
    • 连接和调用外部扩展功能和系统

5. MSF模块

  • /usr/share/metasploit-framework/modules/
  • 技术功能模块（非流程模块）
    • Exploit：利用系统漏洞进行攻击的动作，此模块对应每一个具体漏洞的攻击方法（主动、被动）
    • Payload：成功exploit之后，真正在目标系统执行的代码或指令
      • Shellcode或系统命令
      • 三种Payload：/usr/share/metasploit-framework/modules/payloads/
      • Single：all-in-one
      • Stager：目标计算机内存有限时，先传输一个较小的payload用于建立连接
      • Stagers：利用stager建立的连接下载的后续payload
      • Stager、Stagers都有多种类型，适用于不同场景
      • Shellcode是payload的一种，由于其建立正向/反向shell而得名
    • Auxiliary：执行信息收集、枚举、指纹探测、扫描等功能的辅助模块（没有payload的exploit模块）
    • Encoders：对payload进行加密，躲避AV检查的模块
    • Nops：提高payload稳定性及维持大小
