1. 发现弱点

  • 发现漏洞
    • 基于端口服务扫描结果版本信息（速度慢）
    • 搜索已公开的漏洞数据库（数量大）
    • 使用弱点扫描器实现漏洞管理
（1）https://www.exploit-db.com Kali 官方公布的漏洞代码
（2）searchsploit 服务类型  查找相关漏洞利用代码 （相干文件放在usr/share/exploitdb/platforms）
（3）Kali自带工具 sandi-gui  ▲2019.12.7暂无此工具

2. 从信息的维度定义漏洞管理

  • 信息收集：
    • 扫描发现网络IP、OS、服务、配置、漏洞
    • 能力要求：定义扫描方式内容和目标
  • 信息管理
    • 格式化信息，并进行筛选、分组、定义优先级
    • 能力要求：资产分组、指定所有者、向所有者报告漏洞
  • 信息输出
    • 向不同层级的人群展示足够的信息量
    • 能力需求：生成报告、导出数据、与SIEM集成

3. 弱点扫描类型

  • 主动扫描
    • 有身份验证
    • 无身份验证
  • 被动扫描
    • 镜像端口抓包
    • 其他来源输入
  • 基于Agent的扫描
    • 支持平台有限

4. 漏洞概念

  • CVSS（Common Vulnerability Scoring System）
    • 通用漏洞评分系统–工业标准
    • 扫描安全漏洞严重程度的统一评分方案
    • V3版本–2015年6月10日
    • Basic Metric：基础的恒定不变的弱点权重
    • Temporal Metric：依赖时间因素的弱点权重
    • Enviromental Metric：利用弱点的环境要求和实施难度的权重
    • CVSS是安全内容自动化协议（SCAP）的一部分
    • 通常CVSS与CVE一同由美国国家漏洞库（NVD）发布并保持数据的更新
    • 分值范围：0–10
    • 不同机构按CVSS分值定义威胁的中、高、低威胁级别
    • CVSS体现弱点的风险，威胁级别（severity）表示弱点风险对企业的影响
    • CVSS分值是工业标准，但威胁级别不是
  • Vulnerability Reference
  • CVE（Common Vulnerability and Exposures）
    • 已公开的信息安全漏洞字典，统一的漏洞编号标准
    • MITRE公司负责维护（非盈利机构）
    • 扫描器的大部分扫描项都对应一个CVE编号
    • 实现不同产商之间信息交换的统一标准
  • CVE发布流程
    • 发现漏洞
    • CAN负责指定CVE ID
    • 发布到CVE List–CVE-2008-4250
    • MITRE负责对内容进行编辑维护

5. 漏洞基本概念

  • 很多产商维护自己的Vulnerability Reference

    • MS
    • MSKB

  • 其他Vulnerability Reference

    • CERT TA08-297A
    • BID 31874
    • IAVM 2008-A-0081
    • OVAL OVAL6093

  • OVAL（Open Vulnerability and Assessment Language）

    • 描述漏洞检测方法的机器可识别语言
    • 详细的描述漏洞检测的技术细节，可导入自动化检测工具实施漏洞检测工具
    • OVAL使用XML语言描述，包含了严密的语法逻辑

  • CCE

    • 描述软件配置缺陷的一种标准化格式
    • 在信息安全风险评估，配置缺陷的检测是一项重要内容，使用CCE可以配置缺陷以标准的方式展现出来，便于配置缺陷评估的可量化操作

  • CPE（Common Product Enumeration）

    • 信息技术产品、系统、软件包的结构化命令规范，分类命名

  • CWE（Common Weakness Enumeration）

    • 常见漏洞类型的字典，描述不同类型漏洞的特征（访问控制、信息泄露、拒绝服务）

  • Security Content Automation Protocal（SCAP）

    • SCAP是一个集合了多种安全标准框架
    • 六个元素：CVE、OVAL、CCE、CPE、CVSS、XCCDF
    • 目的是以标准的方法展示和操作安全数据
    • 由NIST负责维护

  • SCAP主要解决三个问题

    • 实现高层政策法规等到底层实施的落地（如FISMA，ISO27000系列）
    • 将信息安全所涉及的各个要素标准化（如统一漏洞的命名及严重性度量）
    • 将复杂的系统配置检查工作自动化

  • SCAP是当前美国比较成熟的一套信息安全评估标准体系，其标准化、自动化的思想对信息安全行业产生了深远的影响

  • NVD（National Vulnerability Database）

    • 美国政府的漏洞管理标准数据
    • 完全基于SCAP框架
    • 实现自动化漏洞管理、安全测量、合规要求
    • 包含以下库(https://nvd.nist.gov/)：
    1. 安全检查列表
    2. 软件安全漏洞
    3. 配置错误
    4. 产品名称
    5. 影响程度

Kali Linux笔记(一)：渗透环境介绍

发表于 2019-12-09 | 分类于 Kali
字数统计: 636 字 | 阅读时长 ≈ 2 分钟

1. 安全问题的根源

  • 分层（固守层面，在一个固定的圈子里）
    • 软件工程师，软件开发
    • 数据库管理师，数据库管理以及数据库管理软件的安装与实现
    • 运维
  • 只追求功能的实现
  • 最大的危险还是人

2. 安全目标

  • 先于攻击者发现和防止漏洞出现
    • 攻击型安全（攻击手段探测漏洞，主动）
    • 防护型安全（投入巨大，周期很大，安全层度脆碎）

3.渗透测试

  • 尝试挫败安全防御机制，发现系统安全弱点
  • 从攻击者的角度思考，测量安全防护有效性
  • 证明安全问题的存在，而非破坏（深入了解漏洞价值）
  • 道德约束
  • 法律

4.渗透测试标准

  • PETS（http://www.pentest-standard.org）
  • 前期交互交互阶段（渗透测试范围，每次一个子系统的渗透测试，子系统可以为一个应用系统，要进行简单的信息交互）
  • 情报收集阶段（目标系统，邮箱地址，联系人，DNS，ip，公司动态等，对目标系统进行主动的探测，端口探测，主机探测）
  • 威胁建模阶段（轮廓性认识确定一条两条效率最高的途径）
  • 漏洞分析阶段（软件版本是否存在漏洞，软件逆向，形成针对性代码）
  • 渗透攻击阶段（阶段不一定顺利）
  • 后渗透测试阶段（拿到服务器后，进一步的渗透，获取有价值的信息，很重要，危害大）
  • 渗透测试报告

5.渗透测试标准

  • 渗透测试范围
  • 获得授权
  • 渗透测试方法
    • 是否允许社会工程学
    • 是否允许拒绝服务攻击

6.渗透测试误区

  • 扫描器就是一切
    • 忽视业务逻辑重的逻辑（普通员工有获取经历级别的信息的权限）

7.KALI Linux介绍

  • 基于Debian的Linux发行版本
  • 前身是BackTrack，2013年3月发布
  • 用于渗透测试和安全审计
  • 包含600+安全工具
  • FHS标准目录结构（/etc，/bin，/dev，/var等,BT改版的主要原因）
  • 定制内核
  • 支持ARM和手机平台（for ARM树莓派CD卡烧录）
  • 开源免费

8.KALI Linux策略

  • Root策略（大量渗透测试需要root权限，不同于其他Linux平台）
  • 网络服务策略（网络服务默认是关闭的，无自动启动脚本）
  • 更新升级策略（同Debian漏洞升级一样）

1. 信息收集-DNS

    • 域名记录：A（IPV4主机记录）、C name（别名记录）、NS（域名服务器地址对域解析）、MX（邮件服务器地址对域解析）、ptr（通过IP地址反向解析域名）、text（spf反向垃圾邮件域名）、SOA（启始授权）、AAAA（IPV6地址）
DNS客户端 → 本地DNS服务器 → 根域服务器 → com服务器 → example.com服务器
1.递归查询（一轮一轮递归） 2.迭代查询（从顶至下）
缓存DNS服务器本身并不包含DNS记录（常用于本地DNS服务器）
spf 反向垃圾域名服务，判断邮件的域名是否伪造

  • 🔺命令：nslookup 域名

  • 🔺命令：nslookup set type=ptr ip地址 ▲2019.11.27反向域名解析大概率查不到
  • 🔺命令：nslookup server 202.106.0.20（其他ISP域名服务器的ip地址）
网络为智能DNS，选取最近的DNS服务器，不同地址的DNS服务器是不一样的，边缘网络
  • 🔺命令：nslookup set type=any（任何域名）域名

  • DNS追踪：🔺命令：dig +trace 域名（迭代查询）、🔺命令：dig 域名（递归查询）

5. 网路配置-1

  • 临时IP地址（重启后不生效,在临时设置有时间限制，大约30s连接时间）
    • 🔺命令：dhclient eth0（自动获取IP地址）
    • 🔺命令：ifconfig eth0 192.168.100.x/24（指定一个IP地址）
    • 🔺命令：route add default gw 192.168.100.1（添加默认网关，缺省网关）
    • 🔺命令：netstat -pantu（查看IP网关信息）
    • 🔺命令：route add -net 172.16.0.0/24（vlan的ip地址） gw 192.168.100.x eth0（vlan 配置）
    • 🔺命令：echo nameserver 8.8.8.8>/etc/resolv.conf（vi /etc/resolv.conf修改域名服务器）
  • 固定IP地址（重启后生效）
    • cat /etc/network/interfaces
    • auto eth0
    • iface eth0 inet static
    • address 192.168.100.x（本机IP地址）
    • netmask 255.255.255.0（子网掩码）
    • network 192.168.100.0（网络地址，可不配置）
    • broadcast 192.168.100.255（广播地址，可不配置）
    • gateway 192.168.100.x（网关地址）
    • dns-nameservers 192.168.100.x（可写多个）
    • uproute add-net 172.16.2.0/24 gw 192.168.100.x eth1 （添加内部地址，静态路由）
    • down route del-net 172.168.2.x/24（电脑关闭的时候，关闭静态理由）

13. 并发线程限制

  • ulimit用于限制当前shell内进程的资源使用
  • 查看默认值
    • 🔺命令：ulimit -a
  • 全局配置文件：/etc/security/limits
  • 用途距离
    • 限制堆栈大小：🔺命令：ulimit -s 100（文件大小）
    • 限制shell内存使用：🔺命令：ulimit -m（最大内存大小） 5000 -v（虚拟内存大小） 5000
    • 限制并发次数，openfile ：🔺命令：ulimit -n 900000（次数）
  • 没有直接对socket数量的限制参数
    • Linux系统中一切都是文件，运行中的文件叫做进程
    • 🔺命令：vi /.bashrc→ulimit -n 900000（持久化ulimit）

15. 服务开关

  • Kali Linux默认未启动所有网络服务
  • 🔺命令：update-rc.d ssh defaults（持久化启动服务）
    • 🔺命令：update-rc.d ssh start 20 2 3 4 5 .stop 20 0 1 6.
    • 🔺命令：update-rc.d A defaults 80（启动优先级） 20（关闭优先级）
    • 🔺命令：update-rc.d B defaults 90（启动优先级） 10（关闭优先级）
  • 🔺命令：/etc/init.d/ssh start(临时开启服务)
  • 重启操作系统 🔺命令：init 6
  • 关闭操作系统 🔺命令：init 0
