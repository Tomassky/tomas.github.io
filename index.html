Kali Linux笔记(二十六)：缓冲区溢出介绍

发表于 2019-12-11 | 分类于 Kali

  1. 程序漏洞从哪里来（程序的动态，变量）

    • 罪恶的根源：变量
    • 数据与代码边界不清（输入数据或者代码）
    • 最简漏洞原理–shell脚本漏洞
      • ./a.sh 参数（; | && &）
./a.sh ; pwd →  /root

  1. 缓存区溢出

    • 当缓冲区边界限制不严格时，由于变量传入畸形数据或程序运行错误，导致缓存区“撑爆”，从而覆盖了相邻内存区域的数据；
    • 成功修改内存数据，可造成进程劫持，执行恶意代码，获取服务器控制权限等后果

  2. 如何发现漏洞

    • 源码审计（源码 → 使用了不安全的函数，对用户数据进行过滤，编码）
    • 逆向工程（打包后的程序 → 反汇编 ）
    • 模糊测试（在新的OS环境中，在OS中进行安装，用工具进行监视，调试，插入）
      • 向程序堆栈半随机的数据，根据内存变化判断溢出
      • 数据生成器：生成随机，半随机数据（编程语言实现打向程序）
      • 测试工具：识别溢出漏洞（动态调试工具）

  1. NMAP

    • 所有参数
    • 目标发现
      • -iL （列表文件作为输入）

      • –excludefile （排除文件）

      • -sL（不做扫描，列出地址段）
      • -sn（ping扫描，不做端口扫描）
      • -Pn（彻底扫描地址段，不回包也进行扫描）
      • -PS/PA/PU/PY （TCP SYN/ACK，UDP or SCTP发现）
      • -P0/PS/PE/PP/PM （Ping层的发现/SCTP发现/ICMP echo/时间戳/子网掩码发现）
      • -n/-R （不做DNS解析/自动做反向解析）
      • –dns-servers <serv1[,serv2]>（指定DNS解析服务器）
      • –system-dns（操作系统默认的DNS解析服务器）
      • –traceroute（扫描进行路由追踪）
    • 端口发现
      • sS/sT/sA/sW/sM （TCP SYN/Connect()/ACK/Window/Maimon scans）
      • sU （UDP扫描）
      • sN/sF/sX （TCP Null，Fin and xmas scans）
      • –scanflags （选择要扫描的flags）
      • -sI <zombie host[:probeport]>（僵尸扫描）
      • -sY/sZ （SCTP协议参数）
      • -s0（IP扫描）
      • -b （FTP中继扫描）
    • 指定扫描端口
      • -p U → UDP T → TCP
      • –exclude-ports （排除端口）
      • -F （快速模式，比较少数量端口）
      • -r （连续扫描端口，少于1000个端口）
      • –top-ports （扫描1000端口的前面端口）
      • –port-ratio （扫描更常见的端口）
    • 服务/版本识别
      • -sV （服务特征库匹配）
      • -sV –version-intensity （扫描的深度、强度）
      • -sV –version-light （intensity=2）
      • -sV –version-all （intensity=9）
      • -sV –version-trace（扫描过程进行跟踪）
    • 脚本扫描
      • -sC （脚本扫描，一般是–script=default）
      • –script= （使用Lua脚本）
      • –script-args=<n1=v1[,n2=v2]> （修改参数）
      • –script-update （脚本库更新）
      • –script-trace （进行路由追踪）
      • –script-help= （脚本使用教程帮助）
    • OS 扫描
      • -O （检测OS）
      • –osscan-limit （OS检测限制）
      • –osscan-guess （OS猜测）
    • 时间和性能
      • -‘s’,’m’,’h’,’ms’ （秒、分、时、毫秒）
      • –min-hostgroup/max-hostgroup （并行扫描主机组的数量）
      • –min-parallelism/max-parallelism （并行探测数量）
      • –min-rtt-timeout/max-rtt-timeout/initial-rtt-timeout （rtt时间）
      • –max-retries （最大回包时间）
      • –host-timeout （主机超时时间）
      • –scan-delay/–max-scan-delay （主机延迟发包时间）
      • –min-rate （最小发包速率）
      • –max-rate （最大发包速率）
    • 防火墙/IDS躲避、欺骗
      • -f –mtu （设置传输单元）
      • -D <decoy1,decoy2[,ME]> （伪造源地址，增加噪声IP）
      • -S （欺骗源地址）
      • -e （指定使用的网卡）
      • -g/–source-port （指定源端口进行发包）
      • –proxies <url1,[url2]> （指定代理服务器）
      • –data （指定十六进制字符串，在包的后面增加）
      • –data-string （增加ACSII码字符串）
      • –data-length （数据的长度）
      • –ip-options （修改IP的选项）
      • –ttl （设置IP的TTL值）
      • –spoof-mac <mac address/prefix/vendor name> （欺骗MAC地址）
      • –badsum （发送错误的TCP/UDP/SCTP 校验和）
    • MISC
      • -6 （可以扫IPv6地址）
      • -A （OS，version，script，traceroute的组合命令）
