Kali Linux笔记(七十五)：一端口多服务隧道--sslh

发表于 2020-02-03 | 分类于 Kali
字数统计: 906 字 | 阅读时长 ≈ 3 分钟

1. sslh（ssl+ssh）

  • 端口分配器
    • 根据客户端第一个包检测协议类型
    • 根据协议检测结果将流量转发给不同目标
    • 支持HTTP、HTTPS、SSH、OpenVPN、tinc、XMAPP和可基于正则表达式判断的人和协议类型
    • 适用于防火墙允许443端口入站访问流量的环境

2. 实验

  • Mono防火墙
    • 第一个网卡为Hosy-only方式，第二个网卡为Bridge桥接方式，桥接到无线网络，检查
  • Kali（SSLH）
    • 网卡方式为Host-only方式，检查网络
  • Window Server 2003（内网的CA服务器）
    • 网卡方式为Host-only方式，检查网络
    • 安装web server
      • Win+R → appwiz.cpl → 更改或添加程序 → 应用程序服务器 → IIS → 万维网服务/Active Server Pages，可能需要光盘）
    • 安装CA服务器
      • Win+R → appwiz.cpl → 更改或添加程序 → 证书服务
      • 独立根CA类型、证书名称：LAB CA、有效期限
    • 开始 → Internet信息服务（IIS）管理器
      • w2k3（本地计算机）→ 网站 → 默认网站 → 右键 → 属性
      • 目录安全性 → 服务器证书 → 名称：www.lab.com、2048位长、单位：IT、部门：IT、公用名称：www.lab.com、国家/省市 → 修改申请文件放置位置
      • 浏览器范围访问http://127.0.0.1/certsrv/ → 申请证书 → 高级证书 → BASE64编码 → 粘贴证书服务器代码 → 提交
    • 开始 → 证书服务器管理
      • LAB CA → 挂起的申请 → 选中申请的证书右键 → 所有任务 → 颁发
      • 证书可以在颁发的证书中查看详细信息
    • 浏览器范围访问http://127.0.0.1/certsrv/ → 查看挂起的证书申请状态 → 下载证书 → 保存在桌面
    • 开始 → Internet信息服务（IIS）管理器
      • w2k3（本地计算机）→ 网站 → 默认网站 → 右键 → 属性
      • 目录安全性 → 服务器证书 → 处理挂起的请求并安装证书
      • w2k3（本地计算机）→ 网站 → 默认网站 → 右键 → 属性
      • 目录安全性 → 查看证书
      • w2k3（本地计算机）→ 网站 → 默认网站 → 右键 → 属性
      • 目录安全性 → 编辑 → 要求安全通道（SSL）
      • 停止网站 → 重新启动
  • Kali（SSLH）
    • 浏览器访问https://Window Server 2003的IP，看是否能通过SSL通道进行通信
    • 配置文件：🔺命令：vi /etc/default/sslh
      • RUN=yes
      • DAEMON=/usr/sbin/sslh
      • DAEMON_OPTS=”–user sslh –listen 本机的IP:443 –ssh 127.0.0.1:22 –http 127.0.0.1:80 –ssl Window Server 2003的IP:443 –pidfile /var/run/sslh/sslh.pid”
    • 启动apache服务，修改侦听端口🔺命令：vi /etc/apache2/ports.conf → 127.0.0.1:80，重启apache服务，🔺命令：netstat -pantu查看是否更改
    • 开启SSH服务（修改SSH配置文件）
  • Mono防火墙
    • 浏览器访问mono防火墙
    • 新建NAT规则
      • Interface → WAN
      • External address → Interface address（若为网段其他的IP映射，需设置为WAN address → Server NAT → 加入整个网段）
      • Protocol → TCP
      • External port range → HTTPS/HTTPS
      • NAT IP → KALI的IP
      • Auto-add a firewall rule to permit traffic through this NAT rule
    • 保存应用，Rules的WAN口多出一条rule
  • Kali（SSLH）
    • 🔺命令：service sslh start（netstat -pantu进行查看）
    • 浏览器验证本机http是否正常进行
  • 物理机
    • 浏览器访问http://防火墙WAN的IP:443 → Kali的Web资源
    • 浏览器访问https://防火墙WAN的IP → Window Server 2003的Web资源
    • ssh尝试登陆防火墙WAN的IP:443进行登陆
  • 🔺命令：namp -p443 -sV -A 防火墙WAN的IP
    • 基本可以扫描出所有的服务
  • ▲ 2020.2.3内网sslh代理正常，原因在于访问Kali IP三种方式都能代理，但外网WAN访问防火墙WAN的IP，无法进行访问，进行ping防火墙WAN的IP也无法成功，猜测与防火墙规则有关
