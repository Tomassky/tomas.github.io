1. DNS协议隧道–iodine

  • 基于DNS查询的隧道工具
  • 与同类工具相比的优点
    • 对下行数据不进行编码，因此性能优
    • 支持多平台：Linux、BSD、Mac OS、Windows
    • 最大16个并发连接
    • 强制密码支持
    • 支持同网段隧道IP（不同于服务器、客户端网段）
    • 支持多种DNS记录类型（自动协商使用的DNS记录类型）
    • 丰富的隧道质量检测措施

2. 实验环境（Linux）

  • 实验机子：Linux主机（bodhi）/防火墙（mono）/外网主机（Kali）
  • Kali
    • 网卡方式为Bridge方式，桥接到无线网络，检查网络
  • Window Server 2003（外网的DNS服务器）
    • 网卡方式为Bridge方式，桥接到无线网络，检查网络
    • 配置一个静态的IP地址，设置DNS服务器为127.0.0.1（ipconfig查询所用IP地址）
    • 安装DNS服务器
      • 安装DNS服务器（Win+R → appwiz.cpl → 更改或添加程序 → 网络服务 → 域名系统，可能需要光盘）
      • 开始 → 管理工具 → DNS → 正向查询区域 → 右键 → 新建区域
      • 主要区域、区域名称（lab.com）
    • NS记录指向Kali
      • 右键lab.com空白区域 → 新建主机 → 名称（w2k3）、IP（2003的IP地址）
      • 右键lab.com空白区域 → 新建主机 → 名称（Kali）、IP（Kali的IP地址）
    • 创建区域lab.com，指派二级域test.lab.com
      • lab.com → 右键 → 新建委派 → test → 添加（kali的主机记录）→ 解析
      • 🔺命令：ping kali.lab.com查看是否能进行正常通信
    • 配置转发器（转发到外网的DNS服务器，不做迭代查询）
      • 服务器图标 → 右键 → 属性 → 转发 → 添加物理网络的DNS服务器（ipconfig查询）
      • nslookup解析新浪和百度的域名，查看是否能正常通信
  • Kali
    • 安装squid
      • 🔺命令：apt-get install squid3
  • Mono防火墙
    • 第一个网卡为Hosy-only方式，第二个网卡为Bridge桥接方式，桥接到无线网络，检查
  • bodhi
    • 插入bodhi-3.2.0-64.iso光盘，安装到硬盘，修改网络为Host-only方式，检查网络
    • 查看网管（netstat -nr，看是否为mono网关）
    • 浏览器登陆mono，修改之前TCP/UDP的规则，改为只允许UDP通过，保存修改
  • Window Server 2003（内网的DNS服务器）
    • 网卡方式为Host-only方式，检查网络
    • 配置一个静态的IP地址，设置DNS服务器为127.0.0.1（ipconfig查询所用IP地址）
    • 安装DNS服务器
      • 安装DNS服务器（Win+R → appwiz.cpl → 更改或添加程序 → 网络服务 → 域名系统，可能需要光盘）
      • 开始 → 管理工具 → DNS → 正向查询区域 → 右键 → 新建区域
      • 主要区域、区域名称（lab.com）
    • 配置转发器（转发到外网的DNS服务器，不做迭代查询）
      • 服务器图标 → 右键 → 属性 → 转发 → 外网的DNS服务器（Window Server 2003）
    • 登陆防火墙查看规则（源和目的：Single host or alias，设置目的IP为Window Server 2003的IP地址）
    • nslookup解析lab.com、test.lab.com、新浪、百度的域名，查看是否能正常通信
  • Kali
    • 🔺命令：service squid start
    • 开启SSH服务（擦除以前所作操作）
    • 🔺命令：netstat -pantu（查看3128端口和22端口是否开放）
  • bodhi
    • 🔺命令：sudo vi /etc/resolv.conf → 修改为内网Window Server 2003的IP地址
    • 🔺命令：dig -t ns lab.com（查看是否解析到w2k3.lab.com以及IP地址）
    • 🔺命令：dig -t ns test.lab.com（查看是否正常解析）
    • ▲ 2020.2.1 dig无法正常解析，但nslookup可以正常解析，最后也能正常上网
  • Kali
    • 运行服务器端：🔺命令：iodined -f -c 10.0.0.1（服务器隧道IP） test.lab.com
      • -f 前端显示（可选）
      • -c 不检查客户端IP
      • IP 服务器端隧道IP的地址
    • ▲ 需要设置密码才能建立隧道，ifconfig查看是否多出一个网卡
  • bodhi
    • 通过修改网卡方式为Bridge，桥接网络，进行下载iodine
    • 运行客户端：**`🔺命令：iodine -f test.lab.com（若指定IP，为局域网内DNS的IP地址）
    • ▲ 需要输入密码才能建立隧道（需同服务器端密码相同），ifconfig查看是否多出一个网卡，进行ping 服务器端的IP地址看能否相同
    • 🔺命令：ssh root@10.0.0.1 -p 22进行尝试登陆，看能否能登陆成功
    • 设置浏览器代理，SOCKS proxy → 10.0.0.1:3128，进行网络访问尝试 ▲ 2020.2.2 可抓取到数据包，但无法上网，Kali的3128监听连接都是IPv6
    • 🔺命令：ssh -CfNg -D 7001 root@10.0.0.1 -p 22进行嵌套SSH隧道
    • 在内网的Window Server 2003上修改Internet选项，设置代理服务器socks Proxy代理，访问网络尝试
  • 隧道网络接口
    • 不基于资源的通用隧道，如同本网段内两台相邻的主机
    • 服务器端和客户端生成隧道网络接口eth0
    • 隧道两端接口的IP地址应不同于客户端和服务器端网段
    • 基于此隧道可嵌套其他隧道技术
