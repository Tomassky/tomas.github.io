Kali Linux笔记(六十九)：DNS隧道--dns2tcp

发表于 2020-02-02 | 分类于 Kali
字数统计: 2.2k 字 | 阅读时长 ≈ 9 分钟

1. DNS协议隧道

  • 防火墙禁止TCP出站访问流量
    • SSH隧道、端口转发全部失效
    • 使用基于UDP协议的隧道
    • DNS的工作原理适合用于实现隧道
    • ▲ TCP53–域名服务的冗余更新，域名服务器之间的传输/UDP53–用于域名服务的查询
  • DNS工作原理
    • DNS隧道原理：注册受自己控制的DNS记录

2. Dns2tcp

  • 利用合法的DNS服务器实现DNS隧道
  • C/S（dns2tcpc/dns2tcpd）结构
  • 通过TXT记录加密传输数据（A记录长度有限，前面最大为63个字节）
  • 隧道建立后保持连接
  • 默认记录生存时间TTL值为3秒，由查询的DNS服务器告诉缓存时间（缓存时间越长越好）
  • 域名记录长度是有限的
  • 安装
    • 🔺命令：apt-get install dns2tcp
    • Kali默认安装
  • 资源访问
    • 本地SSH资源
    • 远程http资源
    • http代理（squid，🔺命令：apt-get install squid3）
    • https代理
    • 隧道嵌套
      • 基于SSH资源将
      • 🔺命令：ssh -CfNg root@127.0.0.1 -p 2222 -D 7001
      • XP IE、Firefox使用嵌套的Socks代理上网

3. 实验-1

  • 实验机子：Linux主机（bodhi）/防火墙（mono）/外网主机（Kali）
  • Mono防火墙
    • 第一个网卡为Hosy-only方式，第二个网卡为Bridge桥接方式，桥接到无线网络，检查
  • bodhi
    • 插入bodhi-3.2.0-64.iso光盘，安装到硬盘，修改网络为Host-only方式，检查网络
    • 查看网管（netstat -nr，看是否为mono网关）
    • 浏览器登陆mono，修改之前TCP/UDP的规则，改为只允许UDP通过，保存修改
    • 浏览器登陆mono，开启默认规则以便能上网，下载dns2tcp、wireshark、firefox，下载完成后关闭默认规则
  • Kali
    • 网卡方式为Bridge方式，桥接到无线网络，检查网络
  • Window Server 2003
    • 网卡方式为Bridge方式，桥接到无线网络，检查网络
    • 安装DNS服务器
      • 安装DNS服务器（Win+R → appwiz.cpl → 更改或添加程序 → 网络服务 → 域名系统，可能需要光盘）
      • 配置一个静态的IP地址，设置DNS服务器为127.0.0.1（ipconfig查询所用IP地址）
      • 开始 → 管理工具 → DNS → 正向查询区域 → 右键 → 新建区域
      • 主要区域、区域名称（lab.com）
    • NS记录指向Kali
      • 右键lab.com空白区域 → 新建主机 → 名称（Kali）、IP（Kali的IP地址）
    • 创建区域lab.com，指派二级域test.lab.com
      • lab.com → 右键 → 新建委派 → test → 添加（Kali的主机记录）→ 解析
    • 配置转发器（转发到外网的DNS服务器，不做迭代查询）
      • 服务器图标 → 右键 → 属性 → 转发 → 添加物理网络的DNS服务器（ipconfig查询）
  • bodhi
    • 🔺命令：sudo vi /etc/resolv.conf → 修改为Window Server 2003的IP地址
    • nslookup
      • 🔺命令：set q=ns lab.com（查看是否为Window Server 2003的IP地址）
      • 🔺命令：set q=ns test.lab.com（查看是否为kali.lab.com）
      • 🔺命令：www.baoidu.com（看是否能解析为外网的IP地址）
      • 🔺命令：www.taobao.com（看是否能解析为外网的IP地址）
  • Kali
    • 服务端配置文件
      • 🔺命令：vi /etc/dns2tcpd.conf（修改listen为0.0.0.0，domain为test.lab.com，增加resources：socks:127.0.0.1:22716,http:192.168.0.1:80,https:0.0.0.0:8087，key=pass123）
      • .dns2tcprcd
      • 资源可以是其他地址
    • 开启socks代理，如GoAgent；开启SSH服务（擦除以前所作操作）
    • 启动
      • 🔺命令：dns2tcpd -F -d 1 -f /etc/dns2tcpd.conf
      • -F：前端显示运行状态
      • -d：dubug level 1-3
      • -f：指定配置文件
    • 开启抓包（Wireshark，只抓取dns的数据包）
  • Window Server 2003
    • 装入抓包软件（ComView，可能需要替换exe文件）
    • Rules → IP Addresses → Both（Kali的IP，mono防火墙的IP）
  • bodhi
    • 禁用mono防火墙的规则
    • 启动抓包工具（sudo wireshark，需要进行过滤数据包）
    • 🔺命令：dns2tcpc -c -k pass123 -d 1 -l 2222 -r ssh -z test.lab.com
      • -c：启动压缩，提高访问效率
      • -k：设置的密码
      • -d：dubug level 1-3
      • -l：本地侦听端口，用于连接隧道
      • -r：资源名称（ssh、socks、http、https）
      • -z：使用的域名
    • 🔺命令：ssh root@127.0.0.1 -p 2222
  • bodhi抓包分析、Kali抓包分析、Window Server 2003抓包分析
  • bodhi（http）
    • 🔺命令：dns2tcpc -c -k pass123 -d 1 -l 2222 -r http -z test.lab.com
    • 浏览器访问127.0.0.1:2222，是否被重定向到192.168.1.1
  • bodhi（socks）
    • 🔺命令：dns2tcpc -c -k pass123 -d 1 -l 2222 -r socks -z test.lab.com
    • 浏览器配置socks代理资源，访问Google尝试
  • bodhi（ssh，作网管模式）
    • 🔺命令：dns2tcpc -c -k pass123 -d 1 -l 2222 -r ssh -z test.lab.com
    • 🔺命令：ssh -CfNg root@127.0.0.1 -p 2222 -D 7001
  • Window XP
    • 网卡方式为Host-only方式，检查网络
    • 浏览器 → 工具（Internet选项）→ 连接 → 局域网设置 → 设置代理服务器（bodhi的7001端口）
    • 上网尝试 ▲ 2020.2.2 ssh隧道的网管模式存在Bug，能连接但无法上网
  • bodhi
    • 浏览器配置HTTP proxy的7001端口，进行上网尝试 ▲ 2020.2.2 ssh隧道的网管模式存在Bug，能连接但无法上网

4. 实验-2

  • FW限制只允许内网DNS服务器访问外网指定DNS服务器UDP53端口
  • 内网DNS服务器：安装DNS服务器，配置转发器到外网DNS服务器
  • 实验机子：Linux主机（bodhi）/防火墙（mono）/外网主机（Kali）
  • Kali
    • 网卡方式为Bridge方式，桥接到无线网络，检查网络
  • Window Server 2003（外网的DNS服务器）
    • 网卡方式为Bridge方式，桥接到无线网络，检查网络
    • 配置一个静态的IP地址，设置DNS服务器为127.0.0.1（ipconfig查询所用IP地址）
    • 安装DNS服务器
      • 安装DNS服务器（Win+R → appwiz.cpl → 更改或添加程序 → 网络服务 → 域名系统，可能需要光盘）
      • 开始 → 管理工具 → DNS → 正向查询区域 → 右键 → 新建区域
      • 主要区域、区域名称（lab.com）
    • NS记录指向Kali
      • 右键lab.com空白区域 → 新建主机 → 名称（w2k3）、IP（2003的IP地址）
      • 本机的NS记录进行更新，指向w2k3.lab.com
      • 右键lab.com空白区域 → 新建主机 → 名称（Kali）、IP（Kali的IP地址）
    • 创建区域lab.com，指派二级域test.lab.com
      • lab.com → 右键 → 新建委派 → test → 添加（kali的主机记录）→ 解析
      • ping kali.lab.com查看是否能进行正常通信
    • 配置转发器（转发到外网的DNS服务器，不做迭代查询）
      • 服务器图标 → 右键 → 属性 → 转发 → 添加物理网络的DNS服务器（ipconfig查询）
      • nslookup解析新浪和百度的域名，查看是否能正常通信
  • Kali
    • 服务端配置文件
      • 🔺命令：vi /etc/dns2tcpd.conf（修改listen为0.0.0.0，domain为test.lab.com，增加resources：http:127.0.0.1:3128，key=pass123）
      • .dns2tcprcd
      • 资源可以是其他地址
    • 开启SSH服务（擦除以前所作操作）
    • 启动
      • 🔺命令：dns2tcpd -F -d 1 -f /etc/dns2tcpd.conf
      • -F：前端显示运行状态
      • -d：dubug level 1-3
      • -f：指定配置文件
    • 安装squid
      • 🔺命令：apt-get install squid3
  • Mono防火墙
    • 第一个网卡为Hosy-only方式，第二个网卡为Bridge桥接方式，桥接到无线网络，检查
  • bodhi
    • 插入bodhi-3.2.0-64.iso光盘，安装到硬盘，修改网络为Host-only方式，检查网络
    • 查看网管（netstat -nr，看是否为mono网关）
    • 浏览器登陆mono，修改之前TCP/UDP的规则，改为只允许UDP通过，保存修改
    • 浏览器登陆mono，开启默认规则以便能上网，下载dns2tcp、wireshark、firefox，下载完成后关闭默认规则
  • Window Server 2003（内网的DNS服务器）
    • 网卡方式为Host-only方式，检查网络
    • 配置一个静态的IP地址，设置DNS服务器为127.0.0.1（ipconfig查询所用IP地址）
    • 安装DNS服务器
      • 安装DNS服务器（Win+R → appwiz.cpl → 更改或添加程序 → 网络服务 → 域名系统，可能需要光盘）
    • 配置转发器（转发到外网的DNS服务器，不做迭代查询）
      • 服务器图标 → 右键 → 属性 → 转发 → 外网的DNS服务器（Window Server 2003）
    • 登陆防火墙查看规则（源和目的：Single host or alias，设置目的IP为Window Server 2003的内网外网IP地址）
    • nslookup解析新浪、百度的域名，查看是否能正常通信
  • Kali
    • 🔺命令：service squid start
    • 🔺命令：netstat -pantu（查看3128端口和22端口是否开放）
  • bodhi
    • 🔺命令：sudo vi /etc/resolv.conf → 修改为内网Window Server 2003的IP地址
    • 🔺命令：dig -t ns lab.com（查看是否解析到w2k3.lab.com以及IP地址）
    • 🔺命令：dig -t ns test.lab.com（查看是否正常解析） ▲ 2020.2.1 dig无法正常解析，但nslookup可以正常解析，最后也能正常上网
    • 🔺命令：dns2tcpc -c -k pass123 -d 1 -l 8080 -r http -z test.lab.com
      • -c：启动压缩，提高访问效率
      • -k：设置的密码
      • -d：dubug level 1-3
      • -l：本地侦听端口，用于连接隧道
      • -r：资源名称（ssh、socks、http、https）
      • -z：使用的域名
    • 设置浏览器代理，HTTP proxy → 127.0.0.1:8080，进行网络访问尝试
