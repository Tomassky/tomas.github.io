Kali Linux笔记(二十)：操作系统识别

发表于 2019-12-10 | 分类于 Kali
字数统计: 883 字 | 阅读时长 ≈ 4 分钟

1. 操作系统识别

  • 操作系统识别技术（默认开放的系统开放端口）
    • 种类繁多
    • 好产品采用多种技术组合
  • TTL起始值
    • Windows：128（65-128）
    • Linux/Unix：64（1-64）
    • 某些Unix：255
TTL值劫持路由器，使不同地区的人访问到网页不一样

2. 操作系统识别

  • Python

    • win=”IP地址”
    • linx=”IP地址”
    • aw=sr1(IP(dst=win)/ICMP())
    • al=sr1(IP(dst=linu)/ICMP())
    • if al[IP].ttl<=64:
    • print “host is linux”
    • else:
    • print “host is windows”
  • ./ttl_os.py IP地址

3. 操作系统识别

  • nmap使用多种技术识别操作系统
    • 🔺命令：nmap IP地址 -O
    • 系统服务特征
CPE国际标准化组织进行的标准化操作系统，设备类型分类

4. 操作系统识别

  • 🔺命令：xprobe2 IP地址
  • 结果有误差

5. 操作系统识别

  • 被动操作系统识别
    • IDS
    • 抓包分析
  • 被动扫描（不主动向目标主机发包，部署在网络出口的地方）
  • 🔺命令：p0f
    • 结合ARP地址欺骗识别全网OS

6. 防火墙识别（尽量隐蔽）

  • 通过检查回包，可能识别端口是否经过防火墙过滤
  • 设备多种多样，结果过存在一定误差
（1）防火墙内部对外部的访问连接  （2）防火墙对源地址的过滤
防火墙几种情况：
Send         Response           Type
SYN          NO              
ACK          RST                Filtered

SYN          SYN+ACK/SYN+RST 
ACK          NO                 Filtered

SYN          SYN+ACK/SYN+RST
ACK          RST                Unfiltered/Open

SYN          NO             
ACK          NO                 Closed

7. 防火墙识别

  • Scapy
  • Python
    • 🔺命令：fw_detect.py IP地址 端口 ▲ 2019.12.6 脚本实验不成功，逻辑报错

8. 防火墙识别

  • Nmap有系列防火墙过滤检测功能
  • 🔺命令：nmap IP地址 端口（默认SYN包）
  • 🔺命令：nmap -sA（ACK包） IP地址 端口

9. 负载均衡识别

  • 广域网负载均衡
    • DNS
    • 智能DNS（不同地区不同DNS回应）
  • 服务器负载均衡
    • 基于Web的服务器
  • 组件级负载均衡
  • HTTP-Loadbalancing
    • Nginx
    • Apache
  • 🔺命令：lbd www.baidu.com
  • 🔺命令：lbd mail.163.com

10. WAF识别

  • WEB应用防火墙（基于规则的过滤 → 基于机器学习，特征识别，语法词法的过滤）
  • 🔺命令：wafw00f -l
  • 🔺命令：wafw00f http://www.microsoft.com
  • 🔺命令：nmap www.microsoft.com --script=http-waf-detect.nse
