1. vega（纯界面化）

  • Kali 2.0以上无此软件，需要下载： https://subgraph.com/vega/download/index.en.html

    ▲ 2019.12.24软件需要java 8 openjdk
▲ 2019.12.24在Windows安装运行成功
▲ 2019.12.24在Kali上运行报错

  • JAVA编写的开源Web扫描器

  • 扫描模式

  • 代理模式

  • 爬站、处理表单、注入测试

  • 支持SSL：http://vega/ca.crt（在代理模式下进行访问）

代理模式：
Web Client → Proxy Server（可截断修改） → Web Server

▲ 代理服务器，一般用于手动漏洞挖掘，可代替客户端进行扫描

爬站：
主动扫描阶段：刚开始的手动提交少提交有攻击性代码的内容
代理扫描阶段：通过字典以及代理服务器发送大量请求进行爬站（Fuzz）

vega比较集中关注Web Application方面的漏洞，比较少关注软件，Web Server方面的漏洞，相对比较集中在程序员编写代码方面的漏洞

2. vega基本参数配置

  • Window → Preferences → General
    • SOCKS proxy/External HTTP proxy
    • General → Appearance（界面显示方式，默认）
    • General → Updates（自动更新，默认）
  • Window → Preferences → Proxy（vega自提供代理）
    • Default User-Agent（默认，可删除vega标识）
    • Override client User-Agent（默认）
    • Prevent browser caching（访问服务器每次发送新的请求，默认）
    • Prevent Intermediate（proxy）caching（阻止代理缓存，默认）
    • Listener（vega本机监听网络地址，提供多个代理给局域网）
  • Window → Preferences → Scanner
    • Debug → Log all scanner requests（记录扫描记录，默认）
    • Debug → Display debug output in console（显示详细扫描信息，默认）
▲ vega最主要的是，vega自带的代理（IP地址:端口）
▲ 无论是Proxy还是Scanner，都需要设置Target，Target Scope：Base Paths/Exclude（URL or pattern）
▲ autoproxt设置vega代理模式：IP：端口

3. vega使用

  • Proxy（Target Scope）
  • Scanner（Target Scope+Cookie）
  • Intercept（vega支持很不友好，可能存在bug）
vega是自签名证书，可能会存在别方的地址劫持（别的地方的自签名证书），可以在官方上下载Vega的自签名证书，然后安装到浏览器上
火狐查看证书信息：右键 → View Page Info → Security → View Certificate

▲ 证书颁发是给域名的，不是给IP地址的
