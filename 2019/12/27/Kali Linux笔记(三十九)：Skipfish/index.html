Kali Linux笔记(三十九)：Skipfish

发表于 2019-12-27 | 分类于 Kali
字数统计: 427 字 | 阅读时长 ≈ 1 分钟

1. skipfish

  • C语言编写（Google开发）
  • 实验性的主动web安全评估工具（相比于商业扫描器不遑相让）
  • 基于Web Application进行扫描
  • 递归爬网
  • 基于字典的探测
  • 速度较快
    • 多路单线程，全异步网络I/O，消除内存管理和调度开销
    • 启发式自动内容识别
  • 误报较低

2. skipfish

  • 🔺命令：skipfish -o（存放工作目录） test http://1.1.1.1
  • 🔺命令：skipfish -o test @url.txt
  • 🔺命令：skipfish -o test -S（指定只读字典）complet.wl -W（写成新的字典） a.wl http://1.1.1.1 #字典
  • 参数
    • -I：只检查包含’string’的URL
    • -X：不检查包含’string’的URL #logout
    • -K：不对指定参数进行Fuzz测试
    • -D：跨站点爬取另外一个域（有连接则会跳过去爬取 ）
    • -l：每秒最大请求数
    • -m：每IP最大并发连接数
    • –config：指定配置文件
  • 身份认证（http认证）
    • 🔺命令：skipfish -A（http身份认证信息） user:pass -o test http://1.1.1.1
  • 身份认证（Cookie认证）
    • 🔺命令：skipfish -C（Cookie信息）"name=var" -o test http://1.1.1.1
  • 身份认证参数（表单认证）
    • –auth-form
    • –auth-user-field <表单代码名> （通过form里面的代码名）
    • –auth-user （提交表单的用户名）
    • –auth-pass-field <表单代码名> （通过form里面的代码名）
    • –auth-pass （提交表单的密码）
    • –auth-verify-url （提交成功后显示的界面）
    • –auth-form-target （通过form里面的action确定）
表单代码名可通过查看界面源代码元素去获得，查看表单设置的属性
  • 扫描结束太快
    • 触发了目标站点的连接数限制，降低-m -l数值
