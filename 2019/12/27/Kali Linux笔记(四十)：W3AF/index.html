1. w3af

▲ 2019.12.26 Kali未带此软件，通过github下载安装过于复杂，还未成功
  • Web Application Attack and Audit Framework，基于python语言开发
  • 此框架的目标是帮助你发现和利用所有WEB应用程序漏洞（和msf框架结合）
  • 9大类近150个plugin
    • audit（审计，扫描发现漏洞）
    • infrastructure（基础架构，扫描操作系统，Web Server，是否使用waf）
    • grep（被动扫描插件类型，根据audit扫描发回的信息进行信息搜索）
    • evasion（逃避目标IDS，IBS，WAF的检测）
    • mangle（做一些基于正则表达式的计算）
    • auth（身份认证）
    • bruteforce（暴力破解，fuzz隐藏路径或文件名）
    • output（扫描结果output为一种可被其他工具调用的文件）
    • crawl（爬网，通过超链接）
    • attack（唯一不可以进行人工修改，已经固定）

2. w3af

  • 安装（Kali自带版本执行扫描时挂死）
    • cd ~
    • apt-get update
    • apt-get install -y python-pip w3af
    • pip install –upgrade pip
    • git clone https://github.com/andresriancho/w3af.git
    • cd w3af
    • ./w3af_console（./w3af_gui）
    • apt-get build-dep python-lxml
    • ./tmp/w3af_dependency_install.sh
  • 升级
    • git pull
  • 创建快捷方式
    • cp /usr/share/application/w3af.desktop /root/Desktop
    • gedit w3af.desktop（修改Exec路径）
    • chmod +x w3af
  • 用户接口
    • Console
    • Gui
    • API

4. w3af–身份认证

  • HTTP Basic
  • NTLM
  • Form
  • Cookie（双因素身份认证/anti-CSRF tokens U盘令牌格式）
▲ 安装IIS：运行 → appwiz.cpl → 添加/删除组件 → 应用程序服务器 → IIS
▲ 设置IIS：开始 → 管理工具 → Internet信息服务（IIS）管理器 → 网站 → 默认网站 → 右键 → 属性 → 目录安全性 → 身份验证和访问控制 → 去掉匿名身份认证

5. w3af–基本身份认证

  • 设置HTTP Basic，设置账号密码
  • 配置Configuration
  • 设置Plugin
  • 可通过抓包，follow TCP流，解码Base64得出用户名密码

6. w3af–NTLM身份认证

  • 集成Windows身份认证
  • 配置Configuration
  • 设置Plugin
  • 抓包可看到加密的NTLM信息

7. w3af–Form身份认证

  • 不能配置Configuration
  • 设置Plugin → Auth（detalied/generic）
    • check-string（登陆成功的字符串）
    • data_format（数据源代码，格式）
  • 设置Plugin → crawl → Web_spider → only_forward（限定扫描子目录）
  • 设置Plugin

8. w3af–Cookie身份认证（绕过令牌ID认证的方式）

  • 找到网站的Cookie
  • w3af只接受严格Cookie格式的文件
  • 配置Configuration（严格Cookie格式，需要添加时间，最后加00，用TAB键跳转）
  • 设置Plugin
Cookie文件
.netscape.com    TRUE    /   FALSE   946684799   NETSCAPE_ID 100103
Each line represents a single piece of stored information.A tab is inserted between each of the fields

From left-to-right,here is what each field represents:
domain--The domain that created AND that can read the variable.
flag--A TRUE/FALSE value indicating if all machines within a given domain can access the variable.This value is set automatically by the browser,depending on the value you set for domain
Path--The path within the domain that the variable is valid for.
Secure--A TRUE/FALSE value indicating if a secure connection with the domain is needed to access the variable
Expiration--The UNIX time that variable will expire on.UNIX time is  as the number of seconds since Jan 1,1970 00:00:00 GMT
Name--The name of the variable
Value--The values of the variable

9. w3af–身份认证（截断代理）

  • 截断代理（▲ 手动Web漏洞挖掘最基础步骤，提交数据代码）
  • HTTP header file（另类的身份认证方法）
  • 保存头文件中不变的变量为一个txt文件，用于Configuration
  • Configuration → General → headers_file
  • Proxies the HTTP requests,allowing their modification → History → Send Request to Manual Editor（自编辑）
  • Proxies the HTTP requests,allowing their modification →History → Fuzzy Request（正则表达式表达）
    • $10$（数字）
    • $string.lowercas[:26]$（字母）
    • $[‘spam’,’eggss’]$（查找特定字符串）
    • $[l.strip() for l in file (‘input.txt’).readline()]$
  • Proxies the HTTP requests,allowing their modification →History → Compare（比较请求的结果）
  • Proxies the HTTP requests,allowing their modification →History → Export Request（导出）
  • Proxies the HTTP requests,allowing their modification →History → History → Audit Request with…（调整修改请求内容）
  • Cluster responses（ID视图查看）

10. w3af–截断代理技术

  • w3af不支持客户端技术（Javascript，Flash，Java applet等）
  • 截断代理手动爬网（Proxy）
    • crawl → spider_man（Plugin）
    • output → export_requests（Output插件）（~/output.b64 64编码）
    • http://127.7.7.7/spider_man?terminate（终止spider_man）
  • crawl → import_results（base64）
