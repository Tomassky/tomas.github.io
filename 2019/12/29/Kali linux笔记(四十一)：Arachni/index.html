1. Arachni ▲ 2019.12.29 软件扫描出问题

  • Kali 自带了旧的arachni阉割版
  • 安装
    • http://www.arachni-scanner.com/download/#Linux
    • 🔺命令：tar xvf arachni.tar.gz
    • http://localhost:9292/ （admin@admin.admin/administrator）

2. Arachni

  • Profile
    • Import
    • Export
    • New
  • Dispatcher
    • 🔺命令：./arachni_rpcd --address=127.0.0.1 --port=1111 --nickname=test1
  • Grid
    • 🔺命令：./arachni_rpcd --nickname=test2 --address=127.0.0.1 --neighbour=127.0.0.1:1111
  • Scan

3. Arachni → Scan

  • new scan
    • IP地址、配置文件、描述、共享
    • Advanced options（Distribution/Scheduling）
▲ 一般的扫描器，要么是直连，要么是通过代理扫描，Arachni不一样，进行分布式扫描，进行负载均衡，每次扫描会生成多个Dispatcher，作为Remote，同一时间不同dispatcher可以扫描不同站点
Direct：直接对目标主机进行扫描
Remote：只有一个Dispatcher
Grid：分布式，多个Dispatcher，网格类型

4. Arachni → Users

  • 集成两个账号（Regular user/Administrator）
  • 可以新建一个新的账号

5. Arachni → Dispatched

  • 控制台创建一个新的Dispatched（🔺命令：./arachni_rpcd --address=127.0.0.1 --port=1111 --nickname=test1）
  • Web界面链接一个存在的Dispatched
▲ Dispatched可以工作在远端服务器，也可以工作在本地服务器上

6. Arachni → Scan

  • 通过Web界面链接一个Dispatched
  • 定义一个Scan，可以选择Remote/Guid
  • 在控制台对一个Dispatched加入一个新的Dispatched（🔺命令：./arachni_rpcd --nickname=test2 --address=127.0.0.1 --neighbour=127.0.0.1:1111）
▲ ?-s：大部分情况下，可以通过这一个命令，查看PHP源代码
▲ 在指定加入组的Dispatched时，可以采用链式的形式，不要同时指向同一个Dipatched

7. Arachni → Profile

  • Profiles Name
    • SQL injection
    • Cross-Site Scripting（XSS）
    • Default
  • Import
  • New Profiles
    • Scope（扫描范围）
    • Audit（扫描项目）
    • Input（提交数据选项）（预设输入，建议修改，审计日志很容易发现）
    • HTTP（HTTP选项）
    • Platform fingerprinting（平台系统选择）
    • Ckecks（扫描功能，主动模块和被动模块）
    • Plugins（插件，发出提示等）（WAF_Detector）
    • Session check（判断是否登陆成功）
